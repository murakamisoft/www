コードレビュー観点チェックリスト

Java / Spring Boot開発におけるコードレビューで確認すべき観点を体系化した実務用チェックリストです。バグ防止・設計品質・可読性・パフォーマンス改善に直結するポイントを整理しています。

最終更新日：2026-06-12

目次

  • 1. バグ防止観点
  • 2. 可読性・保守性
  • 3. 設計・アーキテクチャ
  • 4. パフォーマンス
  • 5. まとめ

1. バグ防止観点

  • nullチェックは適切か
  • 例外処理が握りつぶされていないか
  • 境界値（0 / null / 最大値）の考慮があるか
  • DB更新のトランザクション漏れはないか

2. 可読性・保守性

  • メソッドが長すぎないか（目安：30行以内）
  • 責務が1つにまとまっているか
  • 変数名・クラス名が意味を持っているか
  • コメントが「理由」を説明しているか

3. 設計・アーキテクチャ


Controller → Service → Repository の責務分離が守られているか
  • Controllerにビジネスロジックが入っていないか
  • Serviceが肥大化していないか
  • 依存関係が一方向になっているか

4. パフォーマンス

  • N+1問題が発生していないか
  • 不要なループ・クエリがないか
  • 大量データ取得に制限があるか
  • キャッシュの検討がされているか

5. まとめ

  • コードレビューはバグ防止だけでなく設計品質改善の場
  • チェックリスト化すると属人性がなくなる
  • レビュー基準を固定するとチーム速度が上がる
