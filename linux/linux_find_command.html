findコマンドの使い方と検索例

Linuxにおいて、膨大なファイルの中から目的のものを探し出す「find」コマンドは、システム管理や開発環境で必須のツールです。よく使う検索パターンを中心に、実務で役立つテクニックを解説します。

1. findコマンドの基本構文

findコマンドは、指定したディレクトリ以下のファイルを階層的に検索します。 


find [検索場所] [検索オプション]
オプション 内容
-name ファイル名で検索
-type ファイルタイプ（f:ファイル, d:ディレクトリ）で検索
-mtime 更新日時で検索

2. 実践的な検索パターン

2-1. 名前で検索する

特定のファイル名のものを探します。ワイルドカードを使う際は引用符で囲むのがコツです。 


# .log で終わるファイルをカレントディレクトリ以下から検索
find . -name "*.log"

2-2. 種類とサイズで検索する

ディレクトリだけを探す、または容量が大きいファイルを探す際に便利です。 


# 100MB以上のファイルを探す
find / -type f -size +100M

3. 検索結果に対する操作

見つけたファイルに対して、削除や権限変更を一括で実行できます。


# 見つけたファイルを一括削除する（慎重に！）
find . -name "*.tmp" -exec rm -f {} \;

# 見つけたファイルの所有者を一括変更する
find . -name "*.sh" -exec chown user:group {} \;

4. よくあるエラー

Permission deniedエラー


find: ‘./root’: Permission denied

原因と対処法：
一般ユーザーで閲覧権限のないディレクトリにアクセスした際に発生します。エラーを出さないためには、後ろに 2>/dev/null を追加してエラー出力を捨てるのが一般的です。

5. まとめ

  • `find [場所] -name [条件]`で基本検索。
  • `-type f` や `-size` を組み合わせて目的のファイルを絞り込む。
  • `-exec` を使えば、検索と同時に削除や権限変更ができる。
