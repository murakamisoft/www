ポート使用状況を確認する方法

Webサーバーやアプリケーションを起動しようとした際に「Address already in use」というエラーが出て起動できないことはありませんか？これはポートが既に他のプロセスによって占有されていることが原因です。ポートの使用状況を確認し、プロセスを特定する方法を解説します。

1. ポートとネットワークの概念

コンピュータが外部と通信する際の「出入り口」となるのがポート番号です。OS上の各ポートは、特定のアプリケーション（HTTPなら80番など）と紐付いています。

項目 内容
確認が必要な時 サーバー起動失敗時、ポート衝突時
主なツール ss, lsof, netstat

2. ポートの使用状況を確認するコマンド

2-1. ssコマンド（推奨）

現在のLinux環境で最も標準的なツールです。


# リスニング中のポートを数値で表示
sudo ss -tulpn

2-2. lsofコマンド

ポートを使用しているファイルやプロセスを直接特定できます。


# 特定のポート(例:8080)を使っているプロセスを確認
sudo lsof -i :8080

3. 実行中のプロセスを特定する方法

ポートを占有しているPID（プロセスID）がわかったら、詳細情報を確認して必要に応じて停止します。


# PIDからプロセス名を確認
ps -p  -o comm=

# プロセスを停止（強制終了）
kill -9

4. よくあるエラー

Address already in use


Error: listen tcp :8080: bind: address already in use

原因：起動しようとしたポート番号を、既に他のプロセスが使用しています。上記の手順で特定し、競合するプロセスを停止するか、アプリケーションのポート設定を変更してください。

5. まとめ

  • 起動エラー時は`ss -tulpn`でポートの状態を確認する。
  • `lsof -i :ポート番号`で該当プロセスを特定する。
  • 競合している場合は、PIDを確認して`kill`コマンドで終了させる。
