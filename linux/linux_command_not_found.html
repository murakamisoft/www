command not foundエラーの原因と対処法

Linuxでコマンドを入力した際に「command not found（コマンドが見つかりません）」と表示されて実行できない場合の主な原因と、解決のためのチェックリストを解説します。

1. エラー発生の仕組み

Linuxシェルは、ユーザーが入力したコマンド名から実行ファイルを探しに行きます。この際、環境変数「PATH」に指定されたディレクトリ内を順番に検索しますが、どこにも見つからない場合にこのエラーが発生します。

主な原因 詳細
スペルミス 入力したコマンド名が間違っている
未インストール そもそもそのソフトウェアがインストールされていない
PATH設定漏れ 実行ファイルがある場所が環境変数PATHに含まれていない

2. 主な原因と確認ポイント

2-1. スペルを確認する

まずは単純な入力ミスがないか確認します。特にハイフン（-）の有無や大文字小文字の違いに注意してください。

2-2. インストール済みか確認する

パッケージがインストールされていない場合があります。例えば`git`コマンドがない場合は以下のように確認します。 


# パッケージ管理システムで検索してみる（Debian/Ubuntu系）
apt list --installed | grep git

3. PATHの確認方法

「インストールしたはずなのに見つからない」という場合、PATH設定が正しくない可能性が高いです。現在のPATH設定は以下のコマンドで確認できます。 


# 現在のPATHを表示
echo $PATH

# 特定のコマンドがどこにあるか（または存在するか）を確認
which コマンド名

もし実行ファイルがある場所（例: /usr/local/binなど）が含まれていない場合は、~/.bashrc や ~/.bash_profile にPATHを追記する必要があります。

4. よくあるエラー

パスを通し忘れる例


# コマンドを実行
my-custom-script
# 結果：-bash: my-custom-script: command not found

この場合、スクリプトのフルパス（/opt/scripts/my-custom-script）を直接指定して実行できるか確認してください。実行できるなら、そのディレクトリへのPATHを通すことで解決します。

5. まとめ

  • まずはコマンドのスペルミスを疑う。
  • `which`コマンドを使ってファイルが存在するか確認する。
  • PATHが通っていない場合は、設定ファイル（.bashrc等）を編集する。
