tailコマンドでログ監視する方法

Linuxサーバーでアプリケーションの障害が発生した際、真っ先に確認すべきなのがログファイルです。「tail」コマンドを使うと、ファイルの末尾を表示したり、追加された行をリアルタイムで追跡したりすることができます。

1. tailコマンドとは

tailコマンドは、指定されたファイルの末尾部分を標準出力するコマンドです。サーバーログのように、常に新しい情報が追記されるファイルの監視に最適です。

オプション 内容
-f ファイルを監視し、追加分をリアルタイム表示
-n [行数] 末尾から指定した行数を表示（デフォルトは10）

2. リアルタイム監視の手順

サーバー起動後やエラー発生時に、ログをリアルタイムで眺めるには -f (follow) オプションを使用します。 


# ログファイルをリアルタイムで監視する
tail -f /var/log/syslog

この状態でコマンドを実行すると、新しいログが書き込まれるたびに画面に表示されます。終了するには Ctrl + C を押します。

3. 便利な応用例

grepと組み合わせることで、エラー発生時のみ検知するような監視も可能です。 


# エラーという文字が含まれる行だけをリアルタイム表示
tail -f /var/log/app.log | grep "ERROR"

# 末尾100行を表示してから監視を開始
tail -n 100 -f /var/log/app.log

4. よくあるエラー

Permission deniedエラー


tail: cannot open '/var/log/auth.log' for reading: Permission denied

原因と対処法：
ログファイルがroot権限で保護されているため、一般ユーザーでは開けません。sudo を付けて実行してください。
sudo tail -f /var/log/auth.log

5. まとめ

  • `tail -f` でリアルタイムログ監視ができる。
  • `| grep` を繋げば特定のキーワードのみ抽出できる。
  • 権限不足の場合は `sudo` を忘れないようにする。
