Permission deniedの原因と解決方法

Linuxを使用していて遭遇する最も一般的なエラー「Permission denied」。なぜこのエラーが発生するのか、その仕組みと正しい対処法を解説します。

1. Permission deniedとは

「Permission denied」は、実行しようとした操作に対して、現在のユーザーに十分な権限がないことを示すエラーです。Linuxでは「誰が(User/Group/Other)」「何ができるか(Read/Write/Execute)」が厳格に制御されています。

権限種類 意味
r (read) 読み取り権限
w (write) 書き込み・編集・削除権限
x (execute) 実行権限（ディレクトリの場合は移動権限）

2. 原因の確認手順

まず、対象ファイルの所有者と権限を確認します。


ls -l ファイル名

出力結果の左端に表示される `-rwxr-xr--` といった文字列を確認し、自分のユーザーIDやグループが適切な権限を持っているかチェックします。

3. 解決するためのアプローチ

解決には主に「権限を付与する(chmod)」か「管理者権限を使う(sudo)」の2つがあります。 


# 1. 自分が所有者である場合、権限を追加する
chmod u+w ファイル名

# 2. システム設定ファイルなど、管理者権限が必要な場合
sudo コマンド名

4. よくあるエラー

sudoせずにシステム操作


# 例：システムログを直接編集しようとする
vim /var/log/syslog
# 結果：E45: 'readonly' option is set (add ! to override)

原因：rootユーザーしか書き込めないファイルを一般ユーザーで開いているため。対処として`sudo vim /var/log/syslog`と入力し、root権限で実行してください。

5. まとめ

  • まずは`ls -l`で現在のファイル権限を確認する。
  • 必要な操作が「所有者の変更」か「権限の変更」かを見極める。
  • むやみに全てのファイルを777にせず、必要最小限の権限付与を心がける。
