ディスク容量不足の確認と対処法

Linuxサーバーで「No space left on device」エラーが発生した場合、早急な調査が必要です。ディスク容量を圧迫している場所を特定し、安全に空き容量を確保する手順を解説します。

目次

  • 1. ディスク使用量の確認
  • 2. 容量を圧迫している場所の特定

1. ディスク使用量の確認

まずはどのパーティションがいっぱいになっているかを確認します。`df`コマンドを使用します。 


# ディスク使用量を人間が読みやすい形式（-h）で表示
df -h

「Use%」が100%に近い箇所があれば、そこが原因です。

2. 容量を圧迫している場所の特定

圧迫しているディレクトリを特定するため、`du`コマンドで大きなファイルやディレクトリを探します。 


# ルート直下で容量の大きいディレクトリを上位10件表示
du -sh /* 2>/dev/null | sort -rh | head -n 10

これを繰り返すことで、`/var/log`や`/tmp`など、原因となっている具体的なパスまで追い込みます。

3. 空き容量を確保する方法

不要なログファイルやキャッシュを削除して空き容量を確保します。


# ログファイルを削除せずに中身を空にする（プロセスが掴んでいる場合に有効）
truncate -s 0 /var/log/app.log

# 古いパッケージキャッシュを削除
sudo apt clean

4. よくあるエラー

No space left on device

原因：ディスクの物理的な空き容量がない、または「inode（ファイル数制限）」が上限に達しているケースがあります。 


# inodeの使用状況を確認
df -i

空き容量があるのにエラーが出る場合は、大量の小さなファイルが生成されていないか確認してください。

5. まとめ

  • `df -h`でまず全体の状況を把握する。
  • `du -sh`で容量を食っているディレクトリを特定する。
  • ログや一時ファイルが原因の場合は、`truncate`などで安全に削除する。
