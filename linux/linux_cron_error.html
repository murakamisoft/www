cronが動かない原因と対処法

Linuxで定期実行ジョブ（cron）が期待通りに動作しない場合、環境変数やパスの指定に問題があるケースがほとんどです。本記事では、ジョブが動かない原因を切り分け、確実に実行させるための確認手順を解説します。

1. cronが動かない主な原因

cronは対話型ログインシェルとは環境が異なるため、手動で実行できるコマンドがcronでは動かないことがよくあります。

原因 詳細
PATH設定 絶対パスで指定していないためコマンドが見つからない
権限不足 実行ファイルに実行権限がない
環境変数の欠落 シェル特有の環境変数が読み込まれていない

2. 確認手順

2-1. ログを確認する

cronの実行ログを確認することで、ジョブが呼び出されているか判断できます。 


# cronの実行履歴を確認
grep CRON /var/log/syslog
# または
tail -f /var/log/cron

2-2. 実行権限を確認する

実行対象のスクリプトに実行権限があるか確認します。


ls -l /path/to/your/script.sh
# 実行権限(x)がない場合は chmod +x で付与してください

3. サンプルコード

cronでスクリプトを実行する場合、コマンドはすべて絶対パスで記述するのが基本です。また、実行結果をリダイレクトでログ出力しておくとデバッグが容易になります。 


# 正しい記述例 (crontab -e に記述)
* * * * * /usr/bin/python3 /home/user/script.py >> /home/user/cron.log 2>&1

4. よくあるエラー

環境変数がない場合


# cronで実行するとコマンドが見つからない
/bin/sh: command: command not found

対処法：スクリプト内で必要な環境変数を読み込むか、cronの先頭でPATHを指定します。

5. まとめ

  • コマンドやファイルはすべて「絶対パス」で指定する。
  • 標準エラー出力をファイルにリダイレクトしてログを確認する。
  • シェル環境の違いを考慮し、必要な環境変数を読み込ませる。
