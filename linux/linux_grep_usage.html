grepコマンドの使い方まとめ

Linuxにおいて、ファイルやログから特定の文字列を検索するために欠かせない「grep」コマンド。基本的な使い方から、実務で役立つ便利なオプションまでを体系的に解説します。

1. grepコマンドとは

grepは、指定したファイルや標準入力から、特定のパターン（文字列や正規表現）に一致する行を検索し、その行を表示するコマンドです。

主な用途 説明
エラーログの調査 ログファイルから "Error" を抽出する
設定ファイルの確認 特定のプロパティ設定を探す

2. 基本的な使用手順

2-1. 特定の文字列を検索

ファイル内で文字列を検索する最も基本的な形式です。


grep "検索したい文字列" ファイル名

2-2. 複数ファイルから検索

ディレクトリ内の全ファイルから検索したい場合に使用します。


grep "keyword" ./*

3. 便利な検索オプション

実務ではオプションを組み合わせて効率的に検索します。


# 大文字小文字を区別せず検索 (-i)
grep -i "error" log.txt

# 見つかった行番号も表示 (-n)
grep -n "todo" main.c

# 再帰的にディレクトリ内を検索 (-r)
grep -r "function_name" ./src

4. よくあるエラー

検索できない場合


grep: ファイル名: No such file or directory

原因：指定したファイルパスが間違っている、または権限がない場合に発生します。検索対象のファイルが正しい場所にあるか`ls`コマンドで確認してください。

5. まとめ

  • `grep "文字列" ファイル名` で単純検索。
  • `-i`（大文字小文字無視）、`-n`（行番号表示）を使いこなす。
  • `-r`を使えば大規模プロジェクトのコード検索も一瞬。
