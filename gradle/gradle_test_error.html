テスト実行エラーの原因と解決方法

Gradle（Gradle）環境でのテスト失敗は、JUnit設定・依存関係・テストコード不備が主な原因です。

最終更新日：2026-06-18

目次

  • 1. 概要
  • 2. 原因と対処
  • 3. サンプル設定
  • 4. よくあるエラー
  • 5. まとめ

1. 概要

テストエラーは「テストコードの問題」だけでなく「Gradle設定不備」でも発生します。

原因カテゴリ 内容
JUnit設定 JUnit Platform未設定
依存関係 testライブラリ不足
コード アサーション・例外不備

2. 原因と対処方法

2-1. JUnit設定確認


test {
    useJUnitPlatform()
}

これが無いとJUnit5が動作しません。

2-2. 依存関係確認


testImplementation 'org.springframework.boot:spring-boot-starter-test'

2-3. テスト単体実行


./gradlew test

3. サンプルテスト設定


dependencies {
    testImplementation 'org.junit.jupiter:junit-jupiter'
}

test {
    useJUnitPlatform()
}

4. よくあるエラー

No tests found


No tests found for given includes

原因：テストクラス命名ミス or アノテーション不足

JUnit Platform not found


Could not find JUnit Platform

原因：useJUnitPlatform未設定

5. まとめ

  • テスト失敗は設定・依存・コードの3分類
  • まず useJUnitPlatform() を確認
  • 次に testImplementation を確認
