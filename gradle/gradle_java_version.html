Javaバージョン不一致エラーの解決方法

Gradleビルドで発生するJavaバージョン不一致エラーは、JDK・Gradle・Spring Bootのどれかがズレていることが原因です。本記事では実務での切り分け方法と解決手順を整理します。

1. 概要

Javaバージョン不一致エラーは、Gradle（Gradle）が使用するJDKと、実際にプロジェクトが想定しているJavaバージョンがズレているときに発生します。

項目 内容
典型エラー Unsupported class file major version
原因 JDK不一致・Gradle不整合
影響範囲 ビルド失敗・テスト実行不可

2. 原因と確認手順

2-1. Javaバージョン確認


java -version
./gradlew -version

ここで表示されるJavaとGradleのJVMが一致していないとエラーになります。

2-2. JAVA_HOME確認


echo $JAVA_HOME

実務では「インストール済みなのに古いJavaを見ている」ケースが非常に多いです。

2-3. Gradle設定確認


java {
    toolchain {
        languageVersion = JavaLanguageVersion.of(21)
    }
}

3. サンプルGradle設定


plugins {
    id 'java'
    id 'org.springframework.boot' version '3.2.0'
}

java {
    toolchain {
        languageVersion = JavaLanguageVersion.of(21)
    }
}

4. よくあるエラー

Unsupported class file major version


Unsupported class file major version 65

原因：Java 21でビルドされたクラスを古いJDKで実行している

Toolchain mismatch


No matching toolchain found

原因：Gradle toolchain設定とインストールJDKの不一致

5. まとめ

  • Javaバージョン不一致はJDK・Gradle・toolchainのズレ
  • まず java -version と ./gradlew -version を確認
  • toolchain設定で固定すると再発防止できる
