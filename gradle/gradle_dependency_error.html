依存関係が解決できない原因と対処法

Gradleで発生する「依存関係エラー」は、設定ミスではなく構造的な問題が原因のことが多いです。本記事では原因を5分類し、即復旧できる手順を解説します。

1. 概要

Gradle（Gradle）の依存関係エラーは、主に「ライブラリ取得できない」「バージョン競合」「リポジトリ未設定」の3つに分類されます。

分類 内容
リポジトリ MavenCentral未設定・社内Nexus不通
バージョン 依存ライブラリの競合
キャッシュ Gradleキャッシュ破損

2. 原因と対処方法

2-1. repositories設定ミス


repositories {
    mavenCentral()
}

これが無いと依存関係は100%解決されません。

2-2. キャッシュ削除


./gradlew clean
rm -rf ~/.gradle/caches

2-3. バージョン衝突確認


./gradlew dependencies

どのライブラリが競合しているか特定できます。

3. サンプルGradle設定


dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
    implementation 'com.fasterxml.jackson.core:jackson-databind:2.17.0'
    testImplementation 'org.junit.jupiter:junit-jupiter:5.10.0'
}

4. よくあるエラー

Could not find


Could not find org.example:lib:1.0.0

リポジトリ設定不足

Version conflict


Conflicting dependency versions

5. まとめ

  • 依存関係エラーは「リポジトリ・バージョン・キャッシュ」で切り分け
  • まず ./gradlew dependencies で構造確認
  • clean + caches削除は即効性あり
