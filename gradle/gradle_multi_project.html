マルチプロジェクト構成の作り方

Gradle（Gradle）のマルチプロジェクト構成は、大規模開発で必須となる設計手法です。本記事ではモジュール分割の考え方から実装手順まで実務レベルで解説します。

最終更新日：2026-06-18

1. 概要

マルチプロジェクト構成とは、1つのGradleプロジェクトを複数モジュールに分割して管理する方式です。 例えば「API」「ドメイン」「インフラ」を分けることで保守性が大幅に向上します。

項目 内容
用途 中〜大規模システム開発
メリット 責務分離・再利用性向上
注意点 依存関係設計が複雑化

2. 構成手順

2-1. settings.gradle設定


rootProject.name = 'demo'

include 'api'
include 'domain'
include 'infra'

まずモジュール構造を明示的に定義します。

2-2. 依存関係の整理


dependencies {
    implementation project(':domain')
}

モジュール間依存は最小限に抑えるのが重要です。

3. サンプル構成


root
 ├── api
 ├── domain
 ├── infra

基本は3層構造（API・ドメイン・インフラ）が最も実務で使われます。

4. よくあるエラー

① Project not found


Project with path ':domain' could not be found

原因：settings.gradleにinclude漏れ

② Circular dependency


Circular dependency detected

原因：モジュール間の循環依存

5. まとめ

  • マルチプロジェクトは責務分離が本質
  • settings.gradleの設計が最重要
  • 循環依存を避ける構造設計が必須
