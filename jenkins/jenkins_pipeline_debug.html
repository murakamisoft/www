Jenkins Pipelineが動かない原因とデバッグ方法

Jenkins Pipeline（Jenkinsfile）が動かない原因を体系的に整理し、ログ確認・構文チェック・ステージ設計ミスなど実務で頻出する問題のデバッグ方法を解説します。

1. Pipelineが動かない原因概要

Jenkins Pipelineが動作しない原因は大きく分けて「構文エラー」「ステージ設計ミス」「プラグイン不整合」の3つです。

原因カテゴリ 内容
構文エラー Jenkinsfileの書き間違い
設計ミス stage/steps構造の誤り
環境問題 plugin不足・バージョン差異

2. よくある構文エラー

2-1. pipelineブロック不足

最上位にpipeline {}がないケース。

2-2. stageの閉じ忘れ

中括弧の不足が最頻出。

2-3. steps忘れ

stage内にstepsがないと実行されない。

3. デバッグ手順

3-1. Console Output確認

Jenkinsのビルドログが最重要情報源。

3-2. Replay機能

JenkinsのReplay機能で即修正検証が可能。

3-3. 最小構成で検証


pipeline {
  agent any
  stages {
    stage('Test') {
      steps {
        echo 'OK'
      }
    }
  }
}

4. サンプルJenkinsfile


pipeline {
  agent any

  stages {
    stage('Build') {
      steps {
        sh 'echo build'
      }
    }

    stage('Test') {
      steps {
        sh 'echo test'
      }
    }
  }
}

5. よくあるエラー

Missing required context class


java.lang.NoSuchMethodError / MissingContextVariableException

原因：Pipeline構文の誤りまたはプラグイン不足。

unexpected token

Jenkinsfileの括弧ミスや記述順序ミス。

6. まとめ

  • Pipelineエラーは構文・設計・環境の3分類
  • Consoleログが最重要
  • 最小構成で切り分けるのが基本

Jenkins Pipelineは慣れより「構造理解」が重要です。
