1. Docker連携の概要

JenkinsとDockerを連携すると、ビルド・テスト・デプロイをコンテナ単位で実行できるようになります。 環境差分をなくし、再現性の高いCI/CD構成を実現できます。

構成要素 役割
Jenkins CI/CDパイプライン実行
Docker 実行環境のコンテナ化
Pipeline ビルドフロー定義

2. 環境構築手順

2-1. Dockerインストール

まずDockerをインストールします。


sudo apt update
sudo apt install docker.io -y
sudo systemctl start docker

2-2. JenkinsユーザーをDockerグループに追加


sudo usermod -aG docker jenkins
sudo systemctl restart jenkins

3. Jenkins PipelineでのDocker利用

JenkinsfileでDockerを利用する基本構成です。


pipeline {
  agent any

  stages {
    stage('Build') {
      steps {
        sh 'docker build -t my-app .'
      }
    }

    stage('Test') {
      steps {
        sh 'docker run my-app test'
      }
    }
  }
}

4. サンプルコード

Dockerfileの基本例です。


FROM openjdk:21

COPY target/app.jar app.jar

ENTRYPOINT ["java", "-jar", "/app.jar"]

5. よくあるエラー

docker permission denied


Got permission denied while trying to connect to the Docker daemon

原因：Jenkinsユーザーがdockerグループに入っていない。
対処：usermod -aG docker jenkins を実行し再起動。

docker not found

PATHが通っていないかDocker未インストール。

6. まとめ

  • Jenkins×DockerはCI/CDの標準構成
  • 環境差分をなくし再現性を高める
  • Pipeline化で運用が安定する

実務では「Docker + Jenkins Pipeline」が最も一般的な構成です。
