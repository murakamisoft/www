Jenkins構築手順（初心者向け）

Jenkinsを使えば、ビルド・テスト・デプロイを自動化できます。本記事ではローカル環境でのインストールからSpring Bootプロジェクトの実行までを解説します。

1. 概要

Jenkinsは:contentReference[oaicite:0]{index=0}というCI/CDツールで、ソフトウェアのビルド・テスト・デプロイを自動化します。 Java / Spring Boot開発現場ではほぼ標準的に利用されています。

項目 内容
対象 Spring Boot / Javaアプリケーション
CIツール :contentReference[oaicite:1]{index=1}
ビルドツール Maven / Gradle

2. インストール手順

2-1. Javaの確認

JenkinsはJavaが必要です。まずバージョン確認を行います。


java -version

2-2. Jenkins起動（Windows例）

公式サイトからwarファイルをダウンロードして起動します。


java -jar jenkins.war

起動後、ブラウザで http://localhost:8080 にアクセスします。

3. ジョブ作成

新規ジョブを作成し、ソースコードをGitから取得します。

設定ポイント

  • GitリポジトリURL設定
  • ビルド手順にMaven追加
  • ブランチ指定（main/master）

git clone https://example.com/repo.git
mvn clean install

4. ビルド実行

ジョブを保存後「Build Now」を押すとビルドが開始されます。

成功するとコンソールに以下のように表示されます。


BUILD SUCCESS

5. まとめ

  • JenkinsはCI/CD自動化の標準ツール
  • Java環境とGit設定が重要
  • ビルド設定ミスが失敗の主原因
