Jenkinsで環境変数が反映されない原因まとめ

Jenkinsで環境変数がビルドに反映されない問題の原因と対処法を解説。ジョブ設定ミス、シェル環境の違い、Pipelineでの扱いなど実務で頻出するトラブルを体系的に整理します。

1. 概要

Jenkinsでは環境変数が正しく設定されていないと、ビルド失敗や意図しない動作が発生します。 特にジョブ設定とPipelineの違いによるトラブルが多発します。

項目 内容
発生頻度 高い（実務で頻出）
原因範囲 ジョブ設定・シェル・Pipeline
影響 ビルド失敗・デプロイ失敗

2. 主な原因

2-1. ジョブ設定に環境変数が未設定

Jenkinsの「環境変数設定」が空のままになっているケース。

2-2. シェル環境との差異

.bashrcや.zshrcの変数はJenkinsでは読み込まれない。

2-3. Pipelineでの誤った定義

environmentブロックの位置ミスやスコープ不正。

3. 設定方法と対処

3-1. Pipelineでの正しい定義


pipeline {
  agent any

  environment {
    APP_ENV = "production"
    DB_HOST = "localhost"
  }

  stages {
    stage('Build') {
      steps {
        sh 'echo $APP_ENV'
      }
    }
  }
}

3-2. ジョブ設定での確認

「Build Environment」や「Inject environment variables」を確認する。

4. サンプルコード

シェルスクリプトでの確認例。


echo $APP_ENV
printenv

5. よくあるエラー

環境変数が空になる


APP_ENV=

原因：Jenkinsはログインシェルではないため、OSの環境変数を継承しない。

Pipelineで変数が使えない

environmentブロックのスコープ外で参照しているケースが多い。

6. まとめ

  • Jenkinsはシェル環境と異なる
  • environmentブロックで明示的に定義する
  • ジョブ設定とPipelineの違いを理解する

環境変数トラブルはJenkins運用で最も多い問題の一つです。
