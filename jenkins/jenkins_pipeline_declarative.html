1. Declarative Pipelineとは

Declarative PipelineはJenkinsの標準的なパイプライン記法で、構造化された書き方により可読性と保守性を高めた形式です。 CI/CDの現場では最も一般的に使用されます。

項目 内容
特徴 構造化・標準化されたPipeline
用途 CI/CD自動化
メリット 可読性・再利用性が高い

2. 基本構造

Declarative Pipelineは以下の構造で構成されます。


pipeline {
  agent any

  stages {
    stage('Build') {
      steps {
        echo 'build'
      }
    }

    stage('Test') {
      steps {
        echo 'test'
      }
    }
  }
}

「pipeline → agent → stages → stage → steps」という階層構造が基本です。

3. 実務テンプレート

実務では以下の構成が標準です。


pipeline {
  agent any

  environment {
    APP_ENV = "prod"
  }

  stages {

    stage('Checkout') {
      steps {
        git url: 'https://github.com/user/repo.git'
      }
    }

    stage('Build') {
      steps {
        sh 'mvn clean package'
      }
    }

    stage('Test') {
      steps {
        sh 'mvn test'
      }
    }
  }

  post {
    always {
      echo 'finished'
    }
  }
}

4. サンプルコード

最小構成のDeclarative Pipelineです。


pipeline {
  agent any

  stages {
    stage('Hello') {
      steps {
        echo 'Hello Jenkins'
      }
    }
  }
}

5. よくあるエラー

Declarative syntax error


WorkflowScript: unexpected token

原因：stageやstepsの閉じ忘れ、または階層ミス。

missing required section

agentやstagesが不足している場合に発生。

6. まとめ

  • Declarative Pipelineは標準構文
  • 階層構造の理解が最重要
  • 実務ではenvironment/postが重要

Jenkins CI/CDの基礎として最初に習得すべき構文です。
