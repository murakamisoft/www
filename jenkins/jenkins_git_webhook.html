1. Git連携とWebhookとは

GitHubの変更をトリガーにしてJenkinsが自動でビルドを実行する仕組みがWebhook連携です。 手動実行を不要にし、CI/CDの自動化を実現します。

要素 役割
GitHub ソースコード管理
Webhook 変更通知トリガー
Jenkins ビルド・テスト実行

2. Jenkins側の設定

2-1. Git Plugin確認

Git pluginがインストールされていることを確認。

2-2. Job設定

「Source Code Management」でGitを選択。


Repository URL:
https://github.com/user/repo.git

2-3. Build Trigger設定


GitHub hook trigger for GITScm polling

3. GitHub側のWebhook設定

GitHubリポジトリのSettings → Webhooksから設定します。


Payload URL:
http://your-jenkins-url/github-webhook/

Content type:
application/json

pushイベントを選択することで、自動ビルドが有効になります。

4. サンプル構成

典型的なCI/CD構成です。


pipeline {
  agent any

  stages {
    stage('Checkout') {
      steps {
        git 'https://github.com/user/repo.git'
      }
    }

    stage('Build') {
      steps {
        sh 'mvn clean package'
      }
    }

    stage('Test') {
      steps {
        sh 'mvn test'
      }
    }
  }
}

5. よくあるエラー

Webhookが動かない


404 Not Found / github-webhook

原因：Jenkins URL設定ミスまたはFirewallブロック。

認証エラー

GitHubのAccess Token未設定が原因。

6. まとめ

  • WebhookでCI/CDを自動化できる
  • JenkinsとGitHub連携は実務必須
  • Pipeline化で運用が安定する

Git連携はJenkins運用の入口であり、最重要スキルの一つです。
