Spring BootでBeanが見つからないエラーの原因と解決法

Spring Bootで「NoSuchBeanDefinitionException」が発生する原因と、その具体的な解決方法を実務レベルで解説します。

最終更新日：2026-06-12

目次

  • 1. 概要

1. 概要

Spring Bootでは、DI（依存性注入）によりBeanが自動管理されます。しかし設定ミスや構成漏れがあるとBeanが見つからずエラーになります。

項目 内容
エラー名 NoSuchBeanDefinitionException
原因 ComponentScan漏れ・Bean登録忘れ
影響 アプリ起動失敗・依存解決エラー

2. 主な原因

  • @Component / @Service / @Repository が付いていない
  • @ComponentScanの範囲外にクラスがある
  • 設定クラスがSpringに認識されていない
  • インターフェースのみで実装クラスがない

3. 解決方法

3-1. Componentを付与する


@Service
public class UserService {
}

3-2. ComponentScanを確認


@SpringBootApplication(scanBasePackages = "com.example")
public class App {
}

3-3. Bean定義を明示


@Configuration
public class AppConfig {

  @Bean
  public UserService userService() {
    return new UserService();
  }
}

4. サンプルコード


@RestController
public class UserController {

  private final UserService userService;

  public UserController(UserService userService) {
    this.userService = userService;
  }

  @GetMapping("/user")
  public String getUser() {
    return userService.toString();
  }
}

5. よくあるエラー例


NoSuchBeanDefinitionException: No qualifying bean of type 'xxx' available

このエラーは「Springが対象クラスをBeanとして認識していない」ことが原因です。

6. まとめ

  • Component不足が最頻出原因
  • ComponentScan範囲を必ず確認
  • DI設計の構造ミスが根本原因になることが多い
