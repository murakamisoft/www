Spring Boot起動エラー一覧と解決方法

Spring Bootの起動時に発生する代表的なエラーと、その原因・解決方法を実務レベルで整理しています。

1. 概要

Spring Bootでは依存関係・設定・環境差分などにより、起動時にエラーが発生しやすい。 本記事では代表的な起動エラーと原因切り分けの考え方を整理する。

項目 内容
環境 Java 21
フレームワーク Spring Boot 3.x

2. 起動手順

2-1. Mavenで起動

開発時は以下コマンドで起動する。


mvn spring-boot:run

2-2. Jarで起動

ビルド後にJarを直接実行する。


java -jar target/app.jar

3. サンプルコード

最小構成のController例。


@RestController
public class HelloController {

  @GetMapping("/")
  public String hello() {
    return "Hello Spring Boot";
  }

}

4. よくあるエラー

① Port 8080 already in use

原因：ポート競合。既に別プロセスが8080を使用している。
対処：application.propertiesで変更 


server.port=8081

② Failed to configure DataSource

原因：DB設定不足またはドライバ未設定。
対処：application.ymlのDB設定確認

③ BeanCreationException

原因：DI失敗（Component不足・循環依存）。
対処：@ComponentScan範囲と依存関係を確認

④ Could not resolve placeholder

原因：application.propertiesの値未定義。
対処：${}変数の定義漏れ確認

⑤ Application failed to start

原因：上位例外の結果として発生。
対処：ログのCaused byを必ず追う

5. まとめ

  • 起動エラーは「設定・依存・環境」の3分類で切り分ける
  • 必ずCaused byを起点に原因追跡する
  • ポート・DB・DIが最頻出ポイント
