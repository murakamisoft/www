Spring Boot開発でよくあるエラーと解決集

Spring Boot・MyBatis・Spring Securityを使った開発で遭遇しやすい代表的なエラーと、 原因・解決方法をまとめました。

開発初心者がつまずきやすいエラーを実際のエラーメッセージ付きで解説しています。

1. MyBatisのMapper Beanが見つからない

起動時に以下のようなエラーが発生するケースです。


	Parameter 0 of constructor required a bean of type
	'com.example.repository.BookMapper'
	that could not be found.

原因

  • Mapperインターフェースに@Mapperが付いていない
  • @MapperScanの設定漏れ
  • コンポーネントスキャン対象外のパッケージに配置している

解決方法


	@Mapper
	public interface BookMapper {
	}
または 

	@SpringBootApplication
	@MapperScan("com.example.repository")
	public class Application {
	}

2. Mapper XMLが見つからない

以下のようなエラーが発生します。


	Invalid bound statement (not found)

原因

  • Mapper.xmlのnamespaceが間違っている
  • resources配下に配置されていない
  • id名がインターフェースと一致していない

確認ポイント


	<mapper namespace="com.example.repository.BookMapper">

3. Spring Securityでログイン無限ループ

ログイン画面を表示しても再びログイン画面へ戻されるケースです。

原因

  • ログインURLをpermitAllしていない
  • 認証成功URL設定ミス
  • セッション設定不備

設定例


	http
	  .authorizeHttpRequests(auth -> auth
	      .requestMatchers("/login").permitAll()
	      .anyRequest().authenticated()
	  )
	  .formLogin(form -> form
	      .loginPage("/login")
	      .defaultSuccessUrl("/book")
	      .permitAll()
	  );

4. Whitelabel Error Pageが表示される

アクセス時に真っ白なエラー画面になるケースです。

原因

  • ControllerのURL間違い
  • テンプレート名間違い
  • 404エラー

確認方法


	@GetMapping("/book")
	public String book() {
	    return "book";
	}
templates/book.html が存在することを確認します。

5. Port 8080 was already in use

Spring Boot起動時によく発生します。


	Web server failed to start.
	Port 8080 was already in use.

原因

  • 既に別のSpring Bootが起動している
  • Tomcatが起動中

解決方法

Windowsの場合 

	netstat -ano | findstr 8080
	taskkill /PID プロセス番号 /F
または 

	server.port=8081

6. Thymeleafテンプレートが見つからない


	Error resolving template

原因

  • templatesフォルダ配下に配置されていない
  • ファイル名の誤り
正しい例 

	src/main/resources/templates/book.html

7. JDBC接続エラー


	Communications link failure

原因

  • DBが起動していない
  • URL設定ミス
  • ユーザ名・パスワード誤り

設定例


	spring.datasource.url=jdbc:mysql://localhost:3306/bookmanager
	spring.datasource.username=root
	spring.datasource.password=password

まとめ

Spring Boot開発で発生するエラーの多くは、 設定ファイル・アノテーション・パッケージ構成の確認で解決できます。 まずはエラーログを最後まで読み、原因箇所を特定することが重要です。
