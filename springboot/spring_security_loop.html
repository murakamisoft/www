Spring Bootログイン無限ループの原因

Spring Securityでログイン後に再びログイン画面へ戻される「無限ループ」の原因と対処法を解説します。

1. 概要

Spring Securityでは、認証成功後にリダイレクト設定や権限設定を誤ると、 ログイン画面と認証処理が循環し続ける「無限ループ」が発生します。

項目 内容
環境 Java 21
フレームワーク Spring Boot 3.x / Spring Security 6

2. 主な原因

2-1. ログイン成功後の遷移先が保護されている

認証成功後にリダイレクトされるURLが「認証必須ページ」の場合、再ログイン要求が発生する。

2-2. permitAll設定不足

/login や /error が許可されていないと、認証ループになる。

2-3. CSRFやセッション不整合

CSRFトークン不一致やセッション破棄により再ログイン扱いになる。

3. 設定例と修正方法

典型的な修正例：


@Bean
public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {

    http
        .authorizeHttpRequests(auth -> auth
            .requestMatchers("/login", "/css/**", "/js/**").permitAll()
            .anyRequest().authenticated()
        )
        .formLogin(form -> form
            .loginPage("/login")
            .defaultSuccessUrl("/home", true)
        )
        .logout(logout -> logout
            .logoutSuccessUrl("/login?logout")
        );

    return http.build();
}

ポイントは「ログイン後の遷移先（/home）が必ずアクセス可能であること」。

4. よくあるエラー

無限リダイレクト


ERR_TOO_MANY_REDIRECTS

原因：認証済みでも再度/loginへ戻されるループ状態。

5. まとめ

  • ログイン後遷移先の権限制御を確認する
  • permitAll対象を正しく設定する
  • defaultSuccessUrlの設計ミスを避ける
