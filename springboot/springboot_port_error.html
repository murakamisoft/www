Spring Bootでポート8080が使用中エラーの解決方法

Spring Boot起動時に発生する「ポート8080が使用中」エラーの原因と解決方法を、初心者でも実践できる形で解説します。

1. 概要

Spring Bootはデフォルトで8080ポートを使用しますが、他プロセスと競合すると起動できません。

項目 内容
エラー Port 8080 already in use
原因 他アプリ・過去プロセスの残留
発生タイミング アプリ起動時

2. 原因

  • 別のSpring Bootアプリが起動中
  • IDEのプロセスが残っている
  • Dockerや他サービスが8080使用中
  • 前回プロセスの異常終了

3. 解決方法

3-1. プロセス確認（Windows）


netstat -ano | findstr 8080

3-2. プロセス終了


taskkill /PID xxxx /F

3-3. ポート変更


server.port=8081

4. コマンド例


# Linux / Mac
lsof -i :8080
kill -9 PID

5. よくあるエラー

ポート使用中


Web server failed to start. Port 8080 was already in use.

これはほぼ「既存プロセス残り」が原因です。

6. まとめ

  • まずプロセス確認が最優先
  • 終了できなければポート変更
  • IDE再起動で直るケースも多い
