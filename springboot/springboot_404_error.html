Spring Bootで404エラーが出る原因まとめ

Spring Bootで「Whitelabel Error Page」や404エラーが出る原因を、実務で多い順に整理し、すぐ直せる形で解説します。

1. 概要

Spring Bootの404エラーは「URLとControllerの紐づけが正しくない」場合に発生します。 特に初心者は以下の3点で詰まりやすいです。

項目 内容
エラー 404 Not Found / Whitelabel Error Page
原因 URLマッピング不一致・ComponentScan漏れ
影響 画面・APIが表示されない

2. よくある原因

2-1. URLの打ち間違い

ブラウザURLとControllerのパスが一致していないケースが最も多いです。

2-2. Controllerがスキャンされていない

パッケージ構成がSpring Bootの起動クラス配下にないと認識されません。

2-3. @RequestMapping / @GetMappingの設定ミス

スラッシュの有無や階層ズレで404になります。

2-4. staticとcontrollerの誤解

HTMLを返すつもりがstatic配下に配置されていないケースもあります。

3. 解決手順

3-1. URL確認


http://localhost:8080/xxx

3-2. Controller確認


@RestController
@RequestMapping("/xxx")
public class SampleController {

  @GetMapping
  public String test() {
    return "OK";
  }
}

3-3. パッケージ構成確認

Applicationクラスと同じ階層以下にControllerを配置する。

4. サンプルコード


@RestController
public class HelloController {

  @GetMapping("/hello")
  public String hello() {
    return "Hello Spring Boot";
  }
}

この場合は /hello にアクセスする必要があります。

5. まとめ

  • 404はURLとControllerの不一致が原因
  • パッケージ構成ミスも頻出
  • まずURLとアノテーションを確認する
