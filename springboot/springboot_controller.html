Spring Boot Controllerの作り方

Spring BootでControllerクラスを作成する方法を解説します。@Controllerと@RestControllerの違い、GetMappingの使い方、画面表示までを紹介します。

1. 概要

Controllerはブラウザからのリクエストを受け取り、処理を実行するクラスです。 Spring Bootでは@Controllerまたは@RestControllerアノテーションを利用して作成します。

項目 内容
Java 21
Spring Boot 3.x
用途 HTTPリクエスト受付

2. Controller作成手順

2-1. Controllerクラスを作成する

Controllerクラスを作成し、@Controllerアノテーションを付与します。


@Controller
public class HelloController {

}

2-2. GetMappingを追加する

URLにアクセスしたときに実行するメソッドを作成します。


@Controller
public class HelloController {

    @GetMapping("/")
    public String hello() {
        return "hello";
    }

}

returnで返した文字列はView名になります。 上記の場合はhello.htmlが表示されます。

3. サンプルコード

Thymeleafを利用して画面を表示する例です。


@Controller
public class HelloController {

    @GetMapping("/")
    public String hello(Model model) {

        model.addAttribute(
            "message",
            "Spring Boot Sample");

        return "hello";
    }

}

hello.html


<h1>
  [[${message}]]
</h1>

4. よくあるエラー

Whitelabel Error Page


Whitelabel Error Page

returnしたView名に対応するHTMLファイルが存在しない可能性があります。 src/main/resources/templates配下にHTMLを配置してください。

404 Not Found


404

@GetMappingのURLとアクセス先URLが一致しているか確認してください。

5. まとめ

  • Controllerはリクエストを受け取るクラス
  • @GetMappingでURLとメソッドを紐付ける
  • returnで画面名を返す
  • Modelを利用して画面へ値を渡せる
