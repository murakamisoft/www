Spring Boot REST APIで404・405エラーが出る原因まとめ

Spring Boot REST APIで発生する404（Not Found）・405（Method Not Allowed）エラーの原因と解決方法を実務レベルで解説します。

1. 概要

REST API開発では、URL設計ミスやHTTPメソッドの不一致によって404や405エラーが頻発します。 特にSpring Bootではアノテーション設定ミスが原因になることが多いです。

項目 内容
対象 Spring Boot 3.x
エラー 404 / 405
主な原因 URL・HTTPメソッド・Controller設定

2. 主な原因

  • @RequestMappingのパスミス
  • GET/POSTなどHTTPメソッド不一致
  • Controller未登録
  • ベースパス（context path）違い
  • フロントとバックエンドURL不整合

3. 解決方法

3-1. 正しいGET API


@RestController
@RequestMapping("/api")
public class UserController {

  @GetMapping("/user")
  public String getUser() {
    return "user";
  }
}

3-2. POST API例


@PostMapping("/user")
public String createUser() {
  return "created";
}

4. サンプルコード


fetch("/api/user", {
  method: "GET"
})
.then(res => res.text())
.then(data => console.log(data));

5. よくあるエラー

404 Not Found


Whitelabel Error Page
There was an unexpected error (type=Not Found)

405 Method Not Allowed


Request method 'POST' not supported

ほとんどはURLとHTTPメソッドの不一致が原因です。

6. まとめ

  • URL設計ミスが最頻出
  • HTTPメソッド確認が重要
  • Controllerのパス設計を統一する
