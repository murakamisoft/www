【Spring Boot】Whitelabel Error Pageの原因と対処法

Spring Bootで頻出するWhitelabel Error Pageの原因を整理し、404・マッピングミス・テンプレート不足などの典型パターンと対処法を解説します。

1. 概要

Whitelabel Error Pageは、Spring Bootアプリでエラーページが明示的に設定されていない場合に表示されるデフォルトエラーページです。 主に「URLマッピング不備」や「テンプレート不足」で発生します。

項目 内容
環境 Java 21
フレームワーク Spring Boot 3.x

2. 手順

2-1. 原因の切り分け

まずはログを確認し、404（ルーティングエラー）か500（内部エラー）かを判定します。 


# ログ確認
./mvnw spring-boot:run

2-2. コントローラのマッピング確認

URLに対応するControllerが存在しない場合、Whitelabel Error Pageが表示されます。 


public class SampleController {

  @GetMapping("/hello")
  public String hello() {
    return "hello";
  }

}

2-3. テンプレート不足の確認

Thymeleafなどを使用している場合、HTMLテンプレートが存在しないとエラーになります。

3. サンプルコード

正常に動作する最小構成の例です。


@RestController
public class HelloController {

  @GetMapping("/")
  public String hello() {
    return "Hello Spring Boot";
  }
}

4. よくあるエラー

Whitelabel Error Page


Whitelabel Error Page
This application has no explicit mapping for /error

主な原因は以下です。

  • ControllerのURLマッピングミス（404）
  • Thymeleafテンプレート不足
  • コンテキストパスの誤り
  • Spring Securityによるアクセス制御

対処としては、@GetMappingの確認、テンプレート配置、application.propertiesの設定確認が基本です。

5. まとめ

  • Whitelabel Error PageはSpring Boot標準のエラーページ
  • 多くはURLマッピング不備（404）が原因
  • テンプレートや設定確認でほぼ解決可能
