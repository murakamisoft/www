Spring Boot本番環境で動かない原因（jar・環境差分）

Spring Bootアプリがローカルでは動くのに本番で動かない原因と、その実務的な解決方法を整理します。

1. 概要

Spring Bootは環境差分（ローカル・本番）によって動作が変わる典型的なフレームワークです。 特にjar実行や環境変数の違いでエラーが発生します。

項目 内容
対象 Spring Boot 3.x
環境 Linux / Windows / クラウドサーバ
主要原因 環境差分・jar実行ミス・設定不備

2. 主な原因

  • jarファイルの実行コマンドミス
  • application.propertiesの環境依存設定
  • DB接続情報の差分
  • Javaバージョン違い
  • ポート開放ミス（サーバ側）

3. 解決方法

3-1. 正しいjar実行


java -jar app.jar

3-2. プロファイル指定


java -jar app.jar --spring.profiles.active=prod

3-3. 環境変数の確認


echo $JAVA_HOME
echo $SPRING_PROFILES_ACTIVE

4. デプロイ手順例

  1. ビルド実行（mvn package）
  2. jarファイルをサーバへ転送
  3. java -jarで起動
  4. ログ確認（nohup.out / journalctl）

5. よくあるエラー

ポート使用中


Address already in use

プロファイル未設定


No active profile set

これらはほぼ環境設定ミスが原因です。

6. まとめ

  • ローカルと本番の差分を必ず意識する
  • jar実行とプロファイル指定が重要
  • 環境変数管理が安定稼働の鍵
