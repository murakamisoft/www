Spring BootとMySQL接続エラーの原因と対処法

Spring BootからMySQLへ接続できない原因と解決方法を整理します。ドライバ設定ミスやURL誤りなど、実務で頻出するエラーに対応できるようになります。

1. 概要

Spring BootとMySQLの接続エラーは、設定ミス・ドライバ不足・認証エラーが主な原因です。 特に初期構築時に頻繁に発生します。

項目 内容
対象 Spring Boot 3.x
DB MySQL 8.x
主な原因 URL設定・ドライバ・認証設定

2. 主な原因

  • MySQLドライバ未追加
  • 接続URLの誤り（localhost / port）
  • ユーザー名・パスワードミス
  • MySQLサーバ未起動
  • 認証方式（caching_sha2_password問題）

3. 解決方法

3-1. 依存関係追加


<dependency>
  <groupId>mysql</groupId>
  <artifactId>mysql-connector-j</artifactId>
</dependency>

3-2. application.properties設定


spring.datasource.url=jdbc:mysql://localhost:3306/testdb
spring.datasource.username=root
spring.datasource.password=pass
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver

4. 設定例


spring.datasource.url=jdbc:mysql://127.0.0.1:3306/sampledb?serverTimezone=Asia/Tokyo
spring.datasource.username=root
spring.datasource.password=admin

5. よくあるエラー

接続拒否


Communications link failure

認証エラー


Access denied for user 'root'

これらはほぼ設定ミスかMySQL起動不備です。

6. まとめ

  • ドライバとURL設定が最重要
  • MySQL起動確認を最初に行う
  • 認証方式の違いに注意
