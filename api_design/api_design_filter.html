APIのフィルタ設計ベストプラクティス（検索条件設計）

APIの検索条件設計は「自由にすると壊れる領域」です。本記事ではWHERE条件をAPIに安全に落とす方法と実務パターンを解説します。

1. フィルタ設計とは何か

フィルタ設計とは、データベースのWHERE条件をAPIのリクエストにどう落とすかという設計です。

項目 内容
対象 検索・一覧API
問題 条件が増えるとAPIが破綻する

2. 正しい設計パターン

2-1. クエリパラメータ方式

最も一般的な方法です。


GET /users?name=taro&age=20

2-2. DTOで受ける方式（Spring Boot）


public class UserSearchRequest {
    private String name;
    private Integer age;
}

3. NG設計パターン

NG① SQLをそのままAPIに渡す


/search?query=SELECT * FROM users

→ SQLインジェクションの危険あり（絶対NG）

NG② フリーテキスト乱用

意味不明な検索条件が増えて保守不能になります。

4. Spring Boot実装例


@RestController
@RequestMapping("/users")
public class UserController {

    @GetMapping
    public List search(UserSearchRequest request) {
        return userService.search(request);
    }
}
        

SELECT * FROM users
WHERE (:name IS NULL OR name = :name)
AND (:age IS NULL OR age = :age)

5. よくあるエラー

条件が効かない


WHERE name = NULL

NULL比較ミスが原因です。SQLでは「= NULL」は使えません。

対処法


WHERE name IS NULL

6. まとめ

  • フィルタ設計はAPI設計の核心
  • SQLを直接APIに出さない
  • DTO＋条件分岐が基本形
