APIの冪等性（idempotency）設計とは何か

APIの冪等性とは「同じリクエストを何回送っても結果が変わらない設計」です。決済・登録・リトライ処理で必須となる重要概念です。

1. 冪等性とは何か

冪等性とは「同じ操作を何回実行しても結果が同じになる性質」です。

HTTPメソッド 冪等性
GET 冪等（何回実行しても同じ）
POST 通常は非冪等
PUT 冪等（設計次第で保証可能）

2. なぜ必要か

問題① ネットワークリトライ

通信失敗時に同じリクエストが再送される。

問題② 二重送信

ボタン連打やUI不具合で重複登録が発生する。

問題③ 決済事故

最も危険。二重課金につながる。

3. 正しい設計パターン

3-1. Idempotency Key方式

クライアントから一意キーを送る方式。


Idempotency-Key: 123e4567-e89b

3-2. DBユニーク制約方式


CREATE UNIQUE INDEX idx_order_key ON orders(request_id);

3-3. 状態管理方式

処理済みフラグで重複実行を防止。

4. Spring Boot実装例


@RestController
@RequestMapping("/orders")
public class OrderController {

    @PostMapping
    public String createOrder(
        @RequestHeader("Idempotency-Key") String key
    ) {
        if (orderService.exists(key)) {
            return "ALREADY_PROCESSED";
        }

        orderService.create(key);
        return "OK";
    }
}

冪等キーをDBに保存することで二重処理を防止できます。

5. よくあるエラー

二重登録が発生する


POST /order が2回実行される

原因：冪等性設計がない

対処法

  • Idempotency-Key導入
  • DBユニーク制約
  • 処理済みチェック

6. まとめ

  • 冪等性はAPI安全設計の必須要素
  • POSTは必ず対策が必要
  • 決済・登録系では必須レベル
