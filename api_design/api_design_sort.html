1. ソート設計とは

ソート設計とは、APIの取得結果の並び順を制御する仕組みです。

項目 内容
目的 データの見やすさ・検索性向上
例 created_at DESC, id ASC

2. よくある設計ミス

問題① ユーザー入力をそのままSQLへ

SQLインジェクションリスクが発生する。

問題② ソート項目の未制限

存在しないカラム指定でエラー。

問題③ 複数条件未対応

柔軟性がなく拡張性が低い。

3. 正しい設計パターン

3-1. ホワイトリスト方式

許可された項目のみソート可能にする。

3-2. 複数条件対応

「created_at DESC, id ASC」のように複数指定可能にする。


sort=created_at,desc;id,asc

4. Spring Boot実装例


@GetMapping("/users")
public List getUsers(
    @RequestParam(defaultValue = "created_at") String sort,
    @RequestParam(defaultValue = "desc") String order
) {

    if (!sort.equals("created_at") && !sort.equals("id")) {
        throw new IllegalArgumentException("Invalid sort key");
    }

    return userService.fetch(sort, order);
}

実務ではSpring Data JPAのSortオブジェクトを使うのが一般的です。

5. よくあるエラー

ソートできない

許可リストに存在しないカラム指定。

対処法

  • ホワイトリスト管理
  • Enum化で制御
  • 入力バリデーション追加

6. まとめ

  • ソート設計はUXに直結
  • 必ずホワイトリスト化
  • 複数条件対応が実務標準
