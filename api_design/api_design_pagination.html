REST APIのページング設計（offset vs cursor徹底比較）

大量データを扱うAPIではページング設計が性能とユーザー体験を決定します。本記事ではoffset方式とcursor方式の違いと実務判断を解説します。

1. ページングとは何か

ページングとは、大量データを分割して取得する仕組みです。

方式 特徴
offset ページ番号で取得
cursor 位置情報で取得

2. offset方式の特徴

メリット

  • 実装が簡単
  • ページ番号指定が可能

デメリット

  • データ増加で遅くなる
  • 途中追加でズレる

SELECT * FROM users
LIMIT 10 OFFSET 20;

3. cursor方式の特徴

メリット

  • 高速（インデックス利用）
  • データずれが起きない

デメリット

  • 実装がやや複雑
  • ページ番号指定できない

SELECT * FROM users
WHERE id > :cursor
LIMIT 10;

4. どちらを使うべきか

ケース 推奨方式
管理画面・小規模データ offset
SNS・大量データ cursor

5. Spring Boot実装例


@GetMapping("/users")
public List getUsers(
    @RequestParam int limit,
    @RequestParam(required = false) Long cursor
) {
    return userService.fetch(limit, cursor);
}

6. まとめ

  • ページングはAPI性能の核心
  • 小規模はoffset、大規模はcursor
  • 設計ミスは性能劣化に直結
