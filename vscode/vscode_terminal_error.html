ターミナルが開かない原因と対処法

VSCodeの内蔵ターミナルは開発に欠かせない機能です。突然「ターミナルを作成できませんでした」と表示されたり、起動しても何も表示されない場合の主な原因と解決策を解説します。

最終更新日：2026-06-14

目次

  • 1. ターミナルが開かない主な原因
  • 2. 対処法：設定の確認
  • 3. 対処法：キャッシュとシェルの変更
  • 4. よくあるエラー
  • 5. まとめ

1. 概要

ターミナルが起動しない場合、VSCodeが指定された「既定のシェル（PowerShellやbashなど）」を呼び出せていない可能性が高いです。

主な原因 特徴
設定ミス terminal.integrated.defaultProfileのパス間違い
権限不足 管理者権限の不整合
GPUレンダリング ターミナルの描画がGPUと競合している

2. 対処法：設定の確認

2-1. プロファイルの再設定

コマンドパレット(Ctrl+Shift+P)で「Terminal: Select Default Profile」を実行し、別のシェル（例：Command PromptからPowerShellへ）に切り替えて起動するか試してください。

2-2. GPUアクセラレーションの無効化

描画が原因の場合、settings.jsonに以下を追加することで解決することがあります。 


"terminal.integrated.gpuAcceleration": "off"

3. 対処法：シェルの変更

設定ファイルでデフォルトシェルを明示的に指定します。


{
  "terminal.integrated.profiles.windows": {
    "PowerShell": {
      "source": "PowerShell",
      "args": ["-NoLogo"]
    }
  },
  "terminal.integrated.defaultProfile.windows": "PowerShell"
}

4. よくあるエラー

「The terminal process failed to launch」

シェルのパスが環境変数で正しく定義されていない場合に発生します。OS側の環境変数「Path」を確認してください。

5. まとめ

  • まずは「既定のプロファイルの選択」から別のシェルを試す。
  • それでもダメならGPUアクセラレーションをオフにする。
  • settings.jsonでデフォルトのシェル設定を書き換える。
