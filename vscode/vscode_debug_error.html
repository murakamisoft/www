VSCodeでデバッグができない原因と対処法

VSCodeで「デバッグを開始したのにブレークポイントで止まらない」という問題に直面していませんか？この記事では、原因の特定から解決まで、開発現場で役立つチェックリストを体系的に解説します。

1. デバッグが止まらない主な原因

VSCodeのデバッグ機能が正常に動作しない場合、ソースコードの場所と実行環境の認識がずれていることがほとんどです。特にSpring Boot等のプロジェクトでは、ビルド済みのクラスとソースコードの紐付けが重要になります。

項目 内容
設定ミス launch.jsonのパス指定誤り
環境差異 コンパイル済みの古いクラスが優先されている
拡張機能 Debugger for Java等の不具合・競合

2. 基本的なチェック手順

2-1. ソースコードの配置を確認

VSCodeで開いているワークスペースのルートが正しいか確認してください。ルートが深い階層にあると、正しくデバッガがアタッチされません。 


# プロジェクトのルートディレクトリに移動してVSCodeを開く
code .

2-2. 拡張機能の再読み込み

Java関連の拡張機能がフリーズしていることがあります。コマンドパレット(Ctrl+Shift+P)から「Developer: Reload Window」を実行してみてください。

3. launch.jsonの設定確認

Javaプロジェクトの場合、launch.jsonのmainClassやprojectNameが正しいか再確認しましょう。 


{
  "configurations": [
    {
      "type": "java",
      "name": "Launch Spring Boot",
      "request": "launch",
      "mainClass": "com.example.demo.DemoApplication",
      "projectName": "demo"
    }
  ]
}

4. よくあるエラー

ブレークポイントが中空（灰色）になる

ブレークポイントを置いた場所が灰色になる場合は、コンパイル済みのコードとソースコードの同期が取れていません。一度Maven/Gradleでcleanを行ってから再度起動してください。 


./mvnw clean install

5. まとめ

  • まずはVSCodeを再読み込みする。
  • launch.jsonの設定（メインクラスのパス）を再確認。
  • ビルド成果物が古い可能性があるため、一度クリーンビルドを行う。
