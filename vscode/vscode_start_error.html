VSCodeが起動しない原因と解決方法

VSCodeがアイコンをクリックしても反応しない、あるいはすぐに終了してしまうというトラブルは、バックグラウンドでのプロセス停滞や、破損したキャッシュが原因であることが多いです。この記事で、冷静に対処していきましょう。

目次

1. 起動トラブルの主な要因

起動しない原因は主に以下の3つに分類されます。特に「プロセスの二重起動」や「拡張機能の読み込み失敗」が頻出します。

要因 内容
ゾンビプロセス 前回の終了処理が残っている
拡張機能の競合 アップデート後の不整合
設定破損 settings.json等の不具合

2. 最短で解決する手順

2-1. プロセスの終了

まずはタスクマネージャー（Windows）やアクティビティモニタ（macOS）を開き、Code.exe または Visual Studio Code と名のつくプロセスを全て終了させてください。

2-2. 拡張機能を無効化して起動

コマンドプロンプトやターミナルから、拡張機能を読み込まずに起動するコマンドを試します。 


code --disable-extensions

3. それでも直らない場合

設定ファイル（User Data）を一時的にリネームして、クリーンな状態で起動できるか確認します。 


# Windowsの例：以下のフォルダを一時的にリネーム
%APPDATA%\Code\User

これで起動できた場合、設定ファイルのいずれかが原因です。

4. よくあるエラー

「Executable not found」などのエラー表示

インストールパスが変更されたか、システム上の環境変数が壊れている可能性があります。一度アンインストールし、公式サイトから再インストールするのが確実です。

5. まとめ

  • まずはタスクマネージャーから残ったプロセスを全終了する。
  • --disable-extensionsコマンドで拡張機能の影響を切り分ける。
  • 上記でダメならUser Dataフォルダをリネームしてクリーン起動を試す。
