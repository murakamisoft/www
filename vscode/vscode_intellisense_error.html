IntelliSenseが効かない原因と解決方法

VSCodeの最大の魅力である「IntelliSense（コード補完）」が突然効かなくなると、開発スピードは大きく低下します。本記事では、補完機能が動かない原因の特定方法と、設定の見直し手順を解説します。

1. IntelliSenseが動かない主な原因

コード補完が効かない場合、バックグラウンドで動いている「言語サーバー（Language Server）」がクラッシュしているか、設定ファイルが読み込めていない可能性が高いです。

原因 詳細
言語サーバー 拡張機能のLanguage Serverが応答停止
設定ミス settings.jsonでの補完機能の無効化
メモリ不足 大規模プロジェクトによるインデックス作成の遅延

2. トラブル解決の基本手順

2-1. 言語サーバーの再起動

コマンドパレット(Ctrl+Shift+P)を開き、「Restart Language Server」と入力して実行してください。これでほとんどの補完不具合は解消されます。

2-2. VSCodeのキャッシュクリア

プロジェクトのインデックスが破損している場合は、一度VSCodeを完全に終了し、再起動してください。また、拡張機能の無効化・有効化の切り替えも有効です。

3. 設定ファイルの見直し

settings.jsonで補完機能が無効になっていないか確認しましょう。 


{
  "editor.quickSuggestions": {
    "other": true,
    "comments": false,
    "strings": true
  },
  "editor.parameterHints.enabled": true
}

4. よくあるエラー

補完が極端に遅い（候補が出ない）

不要なファイルが解析対象になっている可能性があります。.vscode/settings.jsonで除外設定を追加しましょう。 


"files.watcherExclude": {
  "**/.git/objects/**": true,
  "**/node_modules/**": true
}

5. まとめ

  • まずはコマンドパレットから言語サーバーを再起動する。
  • settings.jsonで補完設定が有効か確認する。
  • プロジェクトの規模が大きい場合は除外設定でインデックス作成を最適化する。
