VSCode Git連携設定方法まとめ

VSCodeに統合されたGit機能を使うことで、コマンド操作を最小限に抑え、視覚的に変更差分を確認しながら開発できます。本記事では、Gitの初期インストールからVSCodeでの連携設定までを網羅しました。

1. Git連携のメリット

VSCodeをGitと連携させることで、ファイルの変更箇所を色分け表示したり、GUI上でコミットメッセージを入力してそのままリポジトリへ反映させたりと、開発効率が劇的に向上します。

項目 内容
バージョン管理 いつでも過去のコード状態に戻せる
可視化 変更箇所をエディタ内で確認可能
効率化 クリック操作でコミット・プッシュが可能

2. 初期設定手順

2-1. Gitのインストール

まずはPCにGit本体がインストールされているか確認しましょう。コマンドプロンプトで以下のコマンドを打ちます。 


git --version

2-2. VSCodeでのパス設定

GitがインストールされているのにVSCodeが認識しない場合は、settings.jsonで明示的にパスを指定します。 


"git.path": "C:\\Program Files\\Git\\bin\\git.exe"

3. 基本操作（コミット・プッシュ）

VSCodeの左側にある「ソース管理」タブ（枝分かれのアイコン）から、以下の順で操作を行います。

  • 変更のステージング（＋ボタンを押す）
  • コミットメッセージを入力
  • コミットボタンを押下
  • 同期ボタン（プッシュ）でリモートリポジトリへ送信

4. よくあるエラー

認証エラー（認証情報が求められる）

GitHubへのプッシュ時に「Authentication failed」と表示される場合、認証情報が正しく保存されていません。 


# 認証情報をキャッシュさせる設定
git config --global credential.helper store

5. まとめ

  • PC本体にGitをインストールし、パスを通す。
  • VSCodeの「ソース管理」タブを使いこなす。
  • credential.helper設定で認証を効率化する。
