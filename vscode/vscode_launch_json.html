launch.jsonの設定方法

VSCodeでのデバッグ実行を制御する「launch.json」。このファイルを理解しカスタマイズすることで、引数の追加や起動オプションの調整など、複雑なデバッグ環境にも柔軟に対応できるようになります。

1. 概要

launch.jsonは.vscodeフォルダ内に格納される設定ファイルです。デバッグ実行時のプロファイル（構成）を記述し、実行コマンドや環境変数を管理します。

項目 内容
役割 デバッグ実行の構成定義
保存先 プロジェクトルートの .vscode/launch.json
主な用途 起動引数、環境変数、JVMオプションの設定

2. 手順

2-1. ファイルの作成

VSCode左側の「実行とデバッグ」アイコンをクリックし、「launch.jsonファイルを作成します」を選択することで自動生成されます。 


# 生成されたファイルは .vscode/launch.json として保存されます

2-2. 構成の追加

複数の起動パターンが必要な場合は、configurations配列に新しいオブジェクトを追加することで、デバッグメニューから切り替え可能になります。

3. サンプルコード

Spring Bootアプリ起動時にVM引数を指定する設定例です。


{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "java",
      "name": "Launch Spring Boot",
      "request": "launch",
      "mainClass": "com.example.DemoApplication",
      "vmArgs": "-Dserver.port=8081"
    }
  ]
}

4. よくあるエラー

JSONの構文エラー

設定ファイル内でコンマ（,）の付け忘れや、カッコの閉じ忘れがあるとデバッグメニューに構成が表示されません。 


// エラー例：最後の要素に不要なカンマがある
{ "name": "Test", ... , }

ファイル保存時にエラーが出る場合は、JSONのバリデーションを確認してください。

5. まとめ

  • launch.jsonはプロジェクトのデバッグ構成を司る。
  • vmArgsなどを活用して起動オプションを調整する。
  • 構文エラーには注意し、正しくJSONを記述する。
