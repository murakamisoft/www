settings.jsonのおすすめ設定

VSCodeの設定ファイルであるsettings.jsonを適切に編集することで、開発体験は劇的に向上します。保存時の自動フォーマットや、エディタのUI改善など、今すぐ取り入れたい設定を紹介します。

1. 概要

VSCodeの設定はGUIから行えますが、settings.jsonで直接記述すると、設定のバックアップやチーム間での共有が容易になります。

項目 内容
設定形式 JSONフォーマット
利点 設定の高速変更・バックアップ容易性

2. 手順

2-1. ファイルの開き方

コマンドパレット(Ctrl+Shift+P)で「Open User Settings (JSON)」と検索して選択します。

2-2. 設定の適用

開いたJSONファイルに設定したい項目を追記し、保存するだけで即座に反映されます。

3. サンプルコード

開発効率を上げるための代表的な設定例です。


{
  // 保存時に自動整形
  "editor.formatOnSave": true,
  // 文字サイズ
  "editor.fontSize": 14,
  // 行の折り返し
  "editor.wordWrap": "on",
  // 隠しファイル・フォルダの除外
  "files.exclude": {
    "**/.git": true,
    "**/node_modules": true
  }
}

4. よくあるエラー

JSONの文法ミス

最後の行のカンマ忘れなどは、VSCodeがエラーとして警告を出します。JSONとして正しい構造になっているか注意しましょう。 


// エラー：最後の項目の後ろにコンマがあるとJSONエラー
{
  "editor.fontSize": 14,
}

5. まとめ

  • settings.jsonで設定を管理すると共有が楽になる。
  • formatOnSaveは有効にしておくのがおすすめ。
  • 不要なファイルをfiles.excludeで非表示にしてスッキリさせる。
