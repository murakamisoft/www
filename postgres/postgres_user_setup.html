PostgreSQLの初期設定（ユーザー作成・権限設定）

PostgreSQLの実務運用で必須となる「ユーザー作成」「ロール設定」「権限付与」の基本手順を体系的に解説します。

1. PostgreSQL初期設定の全体像

PostgreSQLでは「ロール＝ユーザー」として扱われます。まずロールを作成し、その後データベースと権限を設定します。

  • ロール（ユーザー）作成
  • データベース作成
  • 権限付与

2. ユーザー・ロール作成

基本のユーザー作成：


CREATE ROLE app_user WITH LOGIN PASSWORD 'password';

スーパーユーザー権限付与（開発環境のみ）：


ALTER ROLE app_user WITH SUPERUSER;

3. データベース作成と権限付与

データベース作成：


CREATE DATABASE app_db OWNER app_user;

権限付与：


GRANT ALL PRIVILEGES ON DATABASE app_db TO app_user;

アプリケーション用ユーザーには必要最小限の権限を付与するのが基本です。

4. よくあるエラー

① password authentication failed

ユーザー名またはパスワードが間違っている。

② permission denied for database

権限付与が不足している。

③ role does not exist

ユーザー作成が未実施。

5. まとめ

  • PostgreSQLはロール中心設計
  • ユーザー作成 → DB作成 → 権限付与の順
  • 権限設計が運用の安全性を決める

初期設計を適切に行うことで、後の障害・セキュリティ問題を大幅に減らせます。
