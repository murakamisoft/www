PostgreSQLでパスワード認証エラーの原因と解決方法

PostgreSQLに接続できない原因の多くは「パスワード認証設定ミス」です。本記事ではpg_hba.confの設定ミスやSpring Boot連携時の典型的なエラーを実務レベルで解説します。

1. パスワード認証エラーの概要

PostgreSQLでは、接続時にユーザー認証を行います。このときパスワードが正しくても接続できないケースが多く存在します。

特に以下のようなエラーが頻出します。


FATAL: password authentication failed for user "postgres"

このエラーは「パスワードが間違い」ではなく、設定ミスが原因であることがほとんどです。

2. よくある原因

  • pg_hba.confの認証方式が不適切
  • ユーザー名・パスワードの誤り
  • 接続先ホストの制限（localhostのみ許可など）
  • Spring Bootのapplication.properties設定ミス

3. pg_hba.confの設定確認

PostgreSQLの認証設定はpg_hba.confで管理されています。以下の設定が一般的です。 


# ローカル接続
host    all             all             127.0.0.1/32            md5
host    all             all             ::1/128                 md5

「trust」になっていると認証なし、「md5」または「scram-sha-256」が通常の本番設定です。

4. Spring Boot接続時の注意点

Spring Bootではapplication.propertiesの設定ミスが非常に多いです。


spring.datasource.url=jdbc:postgresql://localhost:5432/testdb
spring.datasource.username=postgres
spring.datasource.password=postgres

特に以下を確認してください。

  • URLのポート番号（5432）
  • DB名の誤り
  • ユーザー権限

5. よくあるエラーと解決法

① password authentication failed

ユーザーまたはpg_hba.confの設定ミス。md5設定を確認。

② no pg_hba.conf entry

接続元IPが許可されていない。接続元を追加する必要あり。

③ connection refused

PostgreSQLが起動していない、またはポートが違う。

6. まとめ

  • パスワードエラーの多くは設定ミス
  • pg_hba.confの確認が最重要
  • Spring Boot側の設定も必ず確認

PostgreSQLの認証エラーは「DB側設定」と「アプリ側設定」の両方を見ることでほぼ解決できます。
