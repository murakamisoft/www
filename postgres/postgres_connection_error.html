Spring Boot × PostgreSQL 接続エラー解決方法

Spring BootからPostgreSQLへ接続する際に発生する代表的なエラーを原因別に整理し、即時解決できるよう実務視点でまとめています。

1. よくある接続エラーの全体像

Spring Boot × PostgreSQL接続で発生するエラーはほぼ以下の3パターンに分類できます。

  • DBサーバ未起動・ポート問題
  • ユーザー認証失敗
  • 設定ミス（URL・ドライバ）

まずはエラーメッセージを「どのカテゴリか」で切り分けるのが最短解決ルートです。

2. Connection refused

エラー例


Connection refused
FATAL: connection to database refused

原因

  • PostgreSQLが起動していない
  • ポート5432が開いていない
  • ホスト設定ミス（localhost/127.0.0.1）

対処


# 起動確認（Linux）
sudo systemctl status postgresql

# 起動
sudo systemctl start postgresql

Windowsの場合は「サービス」からPostgreSQLが起動しているか確認します。

3. 認証エラー

エラー例


FATAL: password authentication failed for user "postgres"

原因

  • ユーザー名・パスワード間違い
  • pg_hba.confの認証方式不一致

対処


-- ユーザー確認
\du

pg_hba.confの「md5 / scram-sha-256」設定を確認してください。

4. JDBC URLミス

エラー例


No suitable driver found
Invalid JDBC URL

原因

  • URLスキーマミス（jdbc:postgresql:// が抜けている）
  • DB名未指定

正しい例


spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/sampledb
    username: postgres
    password: password

5. その他の落とし穴

① ドライバ未追加


org.postgresql:postgresql が未設定

→ Maven / Gradleに依存関係追加が必要

② Docker接続ミス

localhostではなく「host.docker.internal」やコンテナ名を使う必要あり

③ firewall問題

5432ポートが外部からブロックされているケースあり

6. まとめ

  • エラーは3分類で切り分けると最速で解決できる
  • Connection / 認証 / URLの順で確認
  • Docker環境はネットワーク設定が最重要
