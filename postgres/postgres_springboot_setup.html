Spring BootでPostgreSQL接続設定

Spring BootアプリケーションからPostgreSQLへ接続する手順を、依存関係・設定ファイル・疎通確認・エラー対策まで実務レベルで解説します。

1. 概要

Spring BootでPostgreSQLを利用するには、JDBCドライバと設定ファイルの正しい構成が必要です。 本記事では最小構成から実務レベルの設定までを整理します。

項目 内容
環境 Java 21
フレームワーク Spring Boot 3.x
DB PostgreSQL 15+

2. 依存関係設定

PostgreSQL接続にはJDBCドライバを追加します。


<dependency>
  <groupId>org.postgresql</groupId>
  <artifactId>postgresql</artifactId>
  <scope>runtime</scope>
</dependency>

Spring Boot Starter JDBCまたはJPAも必要に応じて追加します。


<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-data-jpa</artifactId>
</dependency>

3. application.yml設定

接続情報を設定します。


spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/sampledb
    username: postgres
    password: password
    driver-class-name: org.postgresql.Driver

  jpa:
    hibernate:
      ddl-auto: update
    show-sql: true

※ 本番環境ではパスワードを直書きせず環境変数化が推奨です。

4. 接続確認サンプル

簡単なREST APIで接続確認を行います。


@RestController
public class DbTestController {

  private final DataSource dataSource;

  public DbTestController(DataSource dataSource) {
    this.dataSource = dataSource;
  }

  @GetMapping("/db-check")
  public String check() throws Exception {
    try (Connection conn = dataSource.getConnection()) {
      return "DB接続成功: " + conn.getMetaData().getDatabaseProductName();
    }
  }
}

5. よくあるエラー

① Connection refused


Connection refused

原因：PostgreSQLが起動していない、またはポートが違う。
対処：5432ポート確認、サービス起動確認。

② password authentication failed


FATAL: password authentication failed for user

原因：ユーザー名・パスワード誤り。
対処：pg_hba.confと認証情報を確認。

6. まとめ

  • 依存関係はpostgresqlドライバが必須
  • application.ymlの設定が最重要
  • 疎通確認用APIを作るとデバッグが早い
