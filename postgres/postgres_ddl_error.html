Spring Boot＋PostgreSQLでDDL自動生成が動かない原因と対処法

Spring BootでEntityを作成したのにテーブルが自動生成されない場合、多くはhibernate ddl-auto設定ミスやEntityスキャン漏れが原因です。本記事では実務で頻出するDDL問題を体系的に解説します。

1. DDL自動生成とは

Spring Boot + JPAでは、Entityクラスから自動的にテーブルを生成できます。この機能がDDL自動生成です。

開発初期には非常に便利ですが、設定ミスがあるとテーブルが作られない原因になります。

2. 動かない主な原因

  • hibernate.ddl-autoが無効（none）
  • Entityクラスに @Entity が付いていない
  • ComponentScan範囲外
  • PostgreSQL接続エラー
  • スキーマ指定ミス

3. application.properties設定

基本設定例：


spring.datasource.url=jdbc:postgresql://localhost:5432/testdb
spring.datasource.username=postgres
spring.datasource.password=postgres

spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true

ポイントは ddl-auto=update です。これがnoneだと何も生成されません。

4. Entity定義の問題

最低限以下が必要です。


@Entity
@Table(name = "sample_table")
public class SampleEntity {

  @Id
  @GeneratedValue(strategy = GenerationType.IDENTITY)
  private Long id;

}

@Entityがない場合、Springはテーブル生成対象として認識しません。

5. よくあるエラー

① Table not found

DDLが実行されていない。ddl-auto設定を確認。

② schema does not exist

スキーマ指定ミス。publicスキーマ確認。

③ Entity not mapped

@Entity漏れまたはComponentScan範囲外。

6. まとめ

  • ddl-auto設定が最重要
  • @Entity漏れが最多原因
  • PostgreSQL接続エラーも連動する

DDL問題は「設定・Entity・DB接続」の3点を順番に確認すればほぼ解決できます。
