Spring BootでPostgreSQLが接続できない（timeout/拒否）原因と対処法

Spring BootからPostgreSQLに接続できないエラーは「DB未起動」「ポート違い」「ネットワーク制限」が主な原因です。本記事では実務で頻出する接続トラブルを体系的に整理します。

1. エラー概要

Spring BootでPostgreSQLに接続できない場合、以下のようなエラーが発生します。 


java.net.ConnectException: Connection refused
java.net.SocketTimeoutException: connect timed out

これらはアプリの問題というより「ネットワーク・DB設定問題」であることがほとんどです。

2. 主な原因

  • PostgreSQLが起動していない
  • ポート5432が違う / 開いていない
  • localhostではなく外部接続が必要
  • pg_hba.confで接続拒否
  • Docker / 仮想環境のネットワーク問題

3. Spring Boot設定確認

まずapplication.propertiesを確認します。


spring.datasource.url=jdbc:postgresql://localhost:5432/testdb
spring.datasource.username=postgres
spring.datasource.password=postgres
spring.datasource.driver-class-name=org.postgresql.Driver

重要ポイント：

  • localhost → Dockerの場合はhost.docker.internal
  • ポート5432が正しいか
  • DB名のスペルミス

4. PostgreSQL側の確認

PostgreSQLが外部接続を許可しているか確認します。


# postgresql.conf
listen_addresses = '*'

pg_hba.confの例：


host    all     all     0.0.0.0/0     md5

設定変更後は必ずPostgreSQLを再起動します。

5. よくあるエラー

① Connection refused

DB未起動またはポート不一致。

② timeout

ネットワーク遮断・Firewall・Docker設定ミス。

③ FATAL: no pg_hba.conf entry

接続元IPが許可されていない。

6. まとめ

  • まずDB起動確認
  • 次にポートと接続先
  • 最後にpg_hba.conf

接続エラーは「アプリではなくインフラ側問題」が大半です。順番に潰すことでほぼ確実に解決できます。
