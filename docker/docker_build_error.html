1. Docker buildエラーとは

Docker buildエラーとは、Dockerイメージ作成時に発生するエラーです。 多くの場合、Dockerfileの記述ミスやファイル配置ミス、ネットワーク障害が原因となります。

特に開発現場では、CI/CD環境やGitHub Actions上でbuildエラーが発生し、 リリース作業が停止するケースも少なくありません。

エラー分類 主な原因
構文エラー Dockerfileの記述ミス
COPYエラー ファイルパス誤り
イメージ取得失敗 Docker Hub接続問題
権限エラー アクセス権不足

2. buildエラーの調査手順

2-1. エラーログを最後まで確認する

Docker buildのエラーメッセージには原因が記載されています。 最初ではなく最後の数行を確認することが重要です。 


docker build -t sample .

2-2. 詳細ログを表示する

BuildKit利用時は詳細ログを確認すると原因調査が容易になります。


docker build --progress=plain .

2-3. Dockerfileの対象ファイルを確認する

COPY対象ファイルが存在するか確認します。


ls -la
tree

3. よくあるDockerfile例

Spring Bootアプリケーションをコンテナ化する例です。


FROM eclipse-temurin:21-jdk

WORKDIR /app

COPY target/app.jar app.jar

EXPOSE 8080

ENTRYPOINT ["java","-jar","app.jar"]

build前にMavenやGradleでjarファイルを生成しておく必要があります。

4. 頻出エラーと対処法

failed to solve: file not found


failed to solve:
failed to compute cache key:
"/target/app.jar" not found

原因

  • jarファイル未作成
  • COPYパス誤り

対処法


mvn clean package

pull access denied


pull access denied for sample-image

原因

  • 存在しないイメージ名
  • Private Repositoryへのアクセス権不足

unknown instruction


unknown instruction: COPPY

原因

  • COPYのスペルミス
  • Dockerfile構文エラー

network timeout


failed to fetch anonymous token
timeout exceeded

原因

  • Docker Hub障害
  • プロキシ設定不備
  • 社内ネットワーク制限

permission denied


permission denied

対処法


sudo docker build .

Linux環境ではdockerグループへユーザー追加も検討してください。

5. まとめ

  • Docker buildエラーの多くはDockerfile記述ミス
  • COPY対象ファイルの存在確認が重要
  • BuildKitの詳細ログを活用する
  • Docker Hub接続状況も確認する
  • エラー全文を確認すると解決が早い
