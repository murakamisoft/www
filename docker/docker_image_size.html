Dockerイメージが大きすぎる原因と最適化方法

Dockerイメージが数GBまで肥大化する原因と、実務で効果の高い軽量化手法を解説します。

最終更新日：2026-06-13

目次

  • 1. Dockerイメージ肥大化の原因
  • 2. 軽量化の具体的方法
  • 3. 実践的なDockerfile例
  • 4. よくある失敗例
  • 5. まとめ

1. Dockerイメージ肥大化の原因

Dockerイメージが大きくなると、ビルド時間・デプロイ時間・レジストリ保存容量・ネットワーク転送量が増加します。 特にCI/CD環境では処理時間増加によるコスト悪化につながります。

原因 内容
不要ファイルをコピー target、node_modules、ログなどを含めている
重いベースイメージ ubuntuやfull JDKを利用している
ビルドツールを残している MavenやGradleを本番イメージに含めている
レイヤーが多い 不要なRUN命令が大量に存在する
キャッシュファイル aptやyumのキャッシュが残っている

Spring Bootプロジェクトでは何も考えずに作成すると1GBを超えるケースも珍しくありません。

2. 軽量化の具体的方法

2-1. .dockerignoreを利用する

不要ファイルをDockerビルド対象から除外します。 もっとも効果が高く、最初に実施すべき対策です。 


target/
.git/
.idea/
*.log
node_modules/

2-2. 軽量ベースイメージを利用する

公式イメージでもサイズは大きく異なります。 Alpine系やJRE専用イメージを利用すると大幅な削減が可能です。 


FROM eclipse-temurin:21-jre-alpine

2-3. Multi-stage Buildを利用する

ビルド用コンテナと実行用コンテナを分離します。 本番環境では最重要のベストプラクティスです。 


FROM maven:3.9-eclipse-temurin-21 AS build

WORKDIR /app

COPY . .

RUN mvn clean package -DskipTests

FROM eclipse-temurin:21-jre-alpine

WORKDIR /app

COPY --from=build /app/target/app.jar app.jar

ENTRYPOINT ["java","-jar","app.jar"]

2-4. キャッシュを削除する

パッケージインストール後にキャッシュを削除します。 不要データを含めないことが重要です。 


RUN apt-get update && \
    apt-get install -y curl && \
    apt-get clean && \
    rm -rf /var/lib/apt/lists/*

3. 実践的なDockerfile例

Spring Bootアプリで実務利用しやすい構成例です。


FROM maven:3.9-eclipse-temurin-21 AS build

WORKDIR /app

COPY pom.xml .
COPY src ./src

RUN mvn clean package -DskipTests

FROM eclipse-temurin:21-jre-alpine

WORKDIR /app

COPY --from=build /app/target/*.jar app.jar

EXPOSE 8080

ENTRYPOINT ["java","-jar","app.jar"]

この構成により、ビルドツールやソースコードを本番イメージに含めずに済みます。

4. よくある失敗例

targetフォルダを丸ごとコピーしている


COPY . .

一見便利ですが、大量の不要ファイルまで取り込む原因になります。 必ず.dockerignoreを設定しましょう。

JDKを本番環境で利用している


FROM eclipse-temurin:21

実行だけならJREで十分です。 JDKを利用すると数百MB増える場合があります。

Multi-stage Buildを使っていない

MavenやGradleがそのまま含まれ、数百MB〜数GBになるケースがあります。

イメージサイズを確認していない


docker images

定期的にサイズを確認し、肥大化を早期発見しましょう。

5. まとめ

  • .dockerignoreで不要ファイルを除外する
  • JREやAlpineベースの軽量イメージを利用する
  • Multi-stage Buildを採用する
  • キャッシュや不要ツールを削除する
  • CI/CD環境ではサイズ監視を実施する
