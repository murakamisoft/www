ポート競合エラー（Port already in use）の解決方法

Docker起動時に発生する「Port already in use」「Bind for 0.0.0.0 failed」などのポート競合エラーの原因と解決方法を解説します。

最終更新日：2026-06-13

目次

  • 1. ポート競合エラーとは
  • 2. 原因調査手順
  • 3. 解決方法
  • 4. よくあるエラーと対処法

1. ポート競合エラーとは

Dockerコンテナ起動時にホスト側で同じポート番号が既に利用されている場合、コンテナは起動できません。

特にSpring Boot（8080）、MySQL（3306）、PostgreSQL（5432）、Redis（6379）で頻繁に発生します。

サービス デフォルトポート
Spring Boot 8080
MySQL 3306
PostgreSQL 5432
Redis 6379
Nginx 80

2. 原因調査手順

2-1. エラーメッセージを確認する


Bind for 0.0.0.0:8080 failed:
port is already allocated

既に8080ポートが利用されていることを示しています。

2-2. 使用中ポートを確認する（Windows）


netstat -ano | findstr :8080

PID（プロセスID）を確認します。


tasklist | findstr PID番号

2-3. 使用中ポートを確認する（Linux/Mac）


lsof -i :8080
または 

netstat -tulpn | grep 8080

2-4. Dockerコンテナ確認


docker ps

以前起動したコンテナが残っている場合があります。

3. 解決方法

方法1: 既存プロセスを停止する

Windows:


taskkill /PID 12345 /F

Linux:


kill -9 12345

方法2: Dockerコンテナを停止する


docker ps


docker stop コンテナ名

不要なコンテナが同じポートを使用している場合があります。

方法3: ポート番号を変更する

Docker実行例:


docker run -p 8081:8080 sample-app

ホスト側8081へ変更しています。

方法4: Docker Compose修正


services:

  app:
    image: sample-app
    ports:
      - "8081:8080"

ホストポートを変更することで競合を回避できます。

Spring Bootでよくある競合

ローカルでSpring Bootを実行したままDocker版も起動すると競合します。 


server.port=8081

開発環境ではポート変更も有効です。

4. よくあるエラーと対処法

port is already allocated


port is already allocated

最も一般的なエラーです。

対処:

  • 利用中プロセス確認
  • Dockerコンテナ停止
  • ポート変更

Bind for 0.0.0.0 failed


Bind for 0.0.0.0:3306 failed

MySQLやPostgreSQLでよく発生します。

ローカルDBが起動中でないか確認してください。

Address already in use


Address already in use

Spring Bootアプリケーション側で発生するエラーです。

server.port設定を変更することで解決できます。

Docker Compose再起動後も解消しない


docker compose down


docker compose up -d

コンテナを完全停止して再作成します。

どのコンテナが使っているかわからない


docker ps --format "table {{.Names}}\t{{.Ports}}"

ポート利用状況を一覧で確認できます。

5. まとめ

  • ポート競合は同じポートを複数プロセスが利用することで発生する
  • まず利用中ポートを特定する
  • docker psで稼働コンテナを確認する
  • 不要なプロセスやコンテナを停止する
  • 必要に応じてポート番号を変更する
