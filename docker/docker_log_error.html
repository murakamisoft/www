Dockerログが見れない原因と対処法

docker logsでログが表示されない、コンテナが停止して確認できない場合の原因と解決方法を解説します。

1. Dockerログが見れない主な原因

Dockerのトラブル調査ではログ確認が最優先です。しかし設定ミスやアプリケーションの実装方法によってはログが表示されないことがあります。

原因 内容
コンテナが停止済み 起動直後に異常終了している
標準出力へ出力していない ファイルログのみになっている
ログドライバ設定 json-file以外を使用している
権限不足 Docker実行権限が不足している
アプリケーション異常終了 ログ出力前にプロセスが終了している

2. ログ確認手順

2-1. コンテナ状態を確認する

まずはコンテナが実行中か確認します。


docker ps -a

STATUS列がExitedになっている場合は異常終了しています。

2-2. コンテナログを確認する


docker logs コンテナ名

例:


docker logs springboot-app

2-3. リアルタイム監視する


docker logs -f springboot-app

アプリケーション起動時の問題調査に有効です。

2-4. 最新100行だけ確認する


docker logs --tail 100 springboot-app

大量ログ環境で便利です。

2-5. コンテナ内部へ接続する


docker exec -it springboot-app sh

アプリケーションログファイルを直接確認できます。

3. Spring Bootでのログ出力設定

Dockerでは標準出力へログを出力するのが基本です。

悪い例:


logging.file.name=/logs/application.log

この場合、docker logsでは確認できません。

推奨:


logging.level.root=INFO

Spring Boot標準設定であればログは標準出力へ出力されます。

Docker Composeの場合


services:
  app:
    image: sample-app

特別な設定をしなければ標準出力がdocker logsに表示されます。

4. よくあるエラーと対処法

docker logs が空になる


docker logs springboot-app

原因:

  • ログ出力がファイルのみ
  • アプリケーションが即終了
  • ログドライバ変更

対処:

  • 標準出力へログ出力する
  • docker inspectで設定確認する
  • コンテナ内部のログファイル確認

No such container


Error: No such container

コンテナ名が間違っています。


docker ps -a

で正しいコンテナ名を確認してください。

permission denied


Got permission denied while trying to connect

Docker実行権限不足です。


sudo docker logs コンテナ名

またはdockerグループへユーザーを追加します。

コンテナが即終了する


docker ps -a

STATUSがExitedになっている場合は起動コマンドやアプリケーションエラーを確認します。

5. まとめ

  • まずdocker ps -aで状態確認する
  • docker logsでログを取得する
  • docker logs -fでリアルタイム監視する
  • Spring Bootは標準出力へログを出す
  • ログが無い場合はコンテナ内部も調査する
