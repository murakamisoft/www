1. DockerとMySQL接続エラーの主な原因

Docker環境ではアプリケーション自体に問題がなくても、ネットワーク設定やコンテナ名の指定ミスによってMySQL接続エラーが発生します。

原因 内容
ホスト名誤り localhostを指定している
MySQL未起動 DBコンテナが停止している
ネットワーク設定ミス 別ネットワークに所属している
認証情報誤り ID・パスワード不一致
初期化中 MySQL起動完了前に接続している

特にSpring Boot初心者が最も多く遭遇するのは「localhost指定ミス」です。

2. 接続確認手順

2-1. コンテナ状態確認


docker ps

MySQLコンテナがUp状態になっていることを確認します。

2-2. ログ確認


docker logs mysql

以下のようなログが表示されれば起動完了です。


MySQL init process done.
ready for connections.

2-3. ネットワーク確認


docker network ls


docker network inspect bridge

アプリケーションコンテナとMySQLコンテナが同じネットワーク上に存在することを確認します。

2-4. コンテナ内部から接続確認


docker exec -it spring-app sh


ping mysql

コンテナ名で名前解決できるか確認します。

3. Spring Boot接続設定例

Docker Compose環境ではlocalhostではなくサービス名を指定します。

Docker Compose


services:

  app:
    build: .

  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: sampledb

application.properties


spring.datasource.url=jdbc:mysql://mysql:3306/sampledb
spring.datasource.username=root
spring.datasource.password=root

「mysql」はDocker Composeのサービス名です。

以下は誤った例です。


spring.datasource.url=jdbc:mysql://localhost:3306/sampledb

コンテナ内のlocalhostは自分自身を指すため、MySQLコンテナへ接続できません。

接続確認用SQL


docker exec -it mysql mysql -u root -p


SHOW DATABASES;

接続できればMySQL自体は正常です。

4. よくあるエラーと対処法

Communications link failure


Communications link failure

原因:

  • MySQL未起動
  • ホスト名誤り
  • ポート誤り

対処:

  • docker psで状態確認
  • サービス名指定へ変更

Connection refused


Connection refused

MySQLプロセスが起動していません。


docker logs mysql

で起動状態を確認します。

Unknown host mysql


Unknown host mysql

ネットワーク設定ミスです。

  • 同じDocker Networkを利用する
  • Compose管理下で起動する

Access denied for user


Access denied for user

認証情報が間違っています。

  • ユーザー名確認
  • パスワード確認
  • 権限確認

Table doesn't exist


Table doesn't exist

接続自体は成功しています。 テーブル作成やFlyway、Liquibaseなどのマイグレーション設定を確認してください。

5. まとめ

  • Docker環境ではlocalhostを使わない
  • サービス名でMySQLへ接続する
  • docker psでコンテナ状態を確認する
  • docker logsでDB起動状況を確認する
  • ネットワーク設定と認証情報を確認する
