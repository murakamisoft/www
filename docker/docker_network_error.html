Dockerコンテナ同士が通信できない、ホスト名で接続できない、Connection refusedになる場合の原因と解決方法を解説します。

1. Dockerネットワークエラーの主な原因

Docker環境ではアプリケーション自体に問題がなくても、コンテナ間通信の設定ミスによって接続エラーが発生します。

原因 内容
異なるNetworkに所属 コンテナ同士が通信できない
ホスト名指定ミス localhostを利用している
コンテナ未起動 接続先サービスが停止中
ポート設定ミス 接続先ポート番号が違う
DNS解決失敗 サービス名が認識されない

Spring BootとMySQL、Redis、NginxなどをDocker Composeで構築した際によく発生するトラブルです。

2. 確認手順

2-1. コンテナ状態を確認する


docker ps

接続先コンテナが起動していることを確認します。

2-2. Network一覧を確認する


docker network ls

Docker Compose利用時は専用ネットワークが自動生成されます。

2-3. Network詳細を確認する


docker network inspect bridge

接続対象コンテナが同じネットワークに存在するか確認します。

2-4. コンテナ内部から疎通確認する


docker exec -it app sh


ping mysql

サービス名で名前解決できるか確認します。

2-5. ポート接続確認


nc -zv mysql 3306

MySQLの3306ポートへ接続できるか確認できます。

3. Docker Composeでの正しい設定例

Docker Composeではコンテナ名ではなくサービス名で通信します。


services:

  app:
    build: .

  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: root

  redis:
    image: redis:7

Spring BootからMySQLへ接続する場合:


spring.datasource.url=jdbc:mysql://mysql:3306/sampledb

Redisへ接続する場合:


spring.data.redis.host=redis

サービス名をホスト名として利用します。

以下は間違った設定です。


spring.datasource.url=jdbc:mysql://localhost:3306/sampledb

コンテナ内のlocalhostはそのコンテナ自身を指します。

ネットワーク作成例


docker network create sample-network

コンテナ起動時にネットワークを指定します。


docker run -d \
  --network sample-network \
  --name mysql \
  mysql:8


docker run -d \
  --network sample-network \
  --name app \
  sample-app

4. よくあるエラーと対処法

Connection refused


Connection refused

原因:

  • 接続先コンテナ停止
  • ポート番号誤り
  • アプリ未起動

対処:

  • docker ps確認
  • docker logs確認

Unknown host


Unknown host mysql

原因:

  • ネットワーク設定ミス
  • サービス名誤り

対処:

  • docker network inspect実施
  • Compose設定確認

No route to host


No route to host

異なるネットワークへ所属している可能性があります。

Temporary failure in name resolution


Temporary failure in name resolution

DNS解決に失敗しています。 コンテナ再作成で解消する場合があります。

通信タイムアウト


Connection timed out

接続先が起動途中、または応答不能になっています。

5. まとめ

  • まずdocker psで起動状態を確認する
  • docker network inspectで所属ネットワークを確認する
  • localhostではなくサービス名を利用する
  • Docker Composeではサービス名で通信する
  • Connection refusedは接続先の起動状態を確認する
