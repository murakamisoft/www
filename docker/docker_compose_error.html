Docker Composeが動かない原因と対処法

Docker Compose実行時に発生するyaml設定ミス、依存関係エラー、 ポート競合、ネットワーク設定問題の原因と解決方法を解説します。

1. Docker Composeが動かない主な原因

Docker Composeが起動しない場合、多くは設定ファイルの記述ミスや コンテナ間の依存関係に原因があります。

原因 発生頻度 内容
YAML記述ミス ★★★★★ スペースやインデント誤り
ポート競合 ★★★★★ 既存プロセスが同じポートを使用
depends_on問題 ★★★★☆ 依存サービス未起動
ネットワーク設定 ★★★☆☆ コンテナ間通信失敗
イメージビルド失敗 ★★★★☆ Dockerfileやbuild設定ミス

2. 原因別の対処方法

2-1. 設定ファイルの文法チェック

YAMLはインデントに非常に厳しいため、 タブ文字やスペース不足で起動できなくなります。 




docker compose config

上記コマンドで構文エラーを確認できます。

2-2. ログを確認する




docker compose logs

docker compose logs -f

Spring BootやMySQLの起動エラーはログから確認できます。

2-3. 強制的に再作成する




docker compose down

docker compose up --build

古いコンテナやキャッシュが原因の場合は再作成で解決することがあります。

2-4. コンテナ状態を確認する




docker ps -a

Exited状態になっているコンテナがないか確認しましょう。

3. 正しいdocker-compose.yml例

Spring BootとMySQLを連携する基本構成です。




services:

app:
build: .
ports:
- "8080:8080"


depends_on:
  - mysql

environment:
  SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/sampledb


mysql:
image: mysql:8.0


environment:
  MYSQL_ROOT_PASSWORD: root
  MYSQL_DATABASE: sampledb

ports:
  - "3306:3306"

localhostではなくサービス名(mysql)を指定するのがポイントです。

4. よくあるエラー集

ERROR: yaml.parser.ParserError




yaml.parser.ParserError

YAMLのインデントが不正です。 スペース数を確認してください。

Bind for 0.0.0.0:8080 failed




Bind for 0.0.0.0:8080 failed:
port is already allocated

8080番ポートが既に利用されています。




netstat -ano | findstr 8080

depends_onで起動失敗




Connection refused

MySQLが完全起動する前にSpring Bootが接続している可能性があります。 ヘルスチェック設定の導入を検討してください。

image build失敗




failed to solve

DockerfileのCOPY先やファイルパスを確認してください。

5. まとめ

  • まず docker compose config で構文チェックする
  • docker compose logs でログを確認する
  • ポート競合を疑う
  • depends_onだけでは起動保証されない
  • コンテナ名ではなくサービス名で通信する
