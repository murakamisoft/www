コンテナ内でSpring Bootが起動しない原因

Dockerコンテナ内でSpring Bootアプリケーションが起動しない場合の原因と対処法を実例付きで解説します。

1. Spring Bootが起動しない主な原因

Spring BootをDocker化した際、ローカルでは動作するのにコンテナ内では起動しないケースがあります。 多くの場合はDockerfile、Javaバージョン、ポート設定、外部接続設定に原因があります。

原因 内容
Jarファイル配置ミス target配下のJarがコピーされていない
Javaバージョン不一致 ビルド環境と実行環境が異なる
ポート設定ミス 8080公開設定が不足
DB接続失敗 MySQL起動前に接続している
環境変数不足 設定値が渡されていない

2. 調査手順

2-1. コンテナログ確認


docker logs コンテナ名

まずはログを確認します。 ほとんどの場合、原因はログに出力されています。

2-2. コンテナ状態確認


docker ps -a

STATUS列がExitedの場合は異常終了しています。

2-3. コンテナ内部確認


docker exec -it コンテナ名 sh

Jarファイルが存在するか確認します。


ls -la

2-4. Javaバージョン確認


java -version

ビルド時と実行時のJavaバージョンが一致しているか確認します。

3. 正しいDockerfile例

Spring Bootで推奨されるDockerfileの例です。


FROM eclipse-temurin:21-jre

WORKDIR /app

COPY target/app.jar app.jar

EXPOSE 8080

ENTRYPOINT ["java","-jar","app.jar"]

Multi-stage Buildを利用する場合:


FROM maven:3.9-eclipse-temurin-21 AS build

WORKDIR /src

COPY . .

RUN mvn clean package -DskipTests

FROM eclipse-temurin:21-jre

WORKDIR /app

COPY --from=build /src/target/*.jar app.jar

ENTRYPOINT ["java","-jar","app.jar"]

本番環境ではMulti-stage Buildの利用を推奨します。

Spring Boot公開設定

コンテナ外からアクセスする場合はポート公開が必要です。


docker run -p 8080:8080 sample-app

Docker Composeの場合:


services:
  app:
    build: .
    ports:
      - "8080:8080"

4. よくあるエラーと対処法

Unable to access jarfile


Unable to access jarfile app.jar

原因:

  • COPY対象誤り
  • Jar名変更

対処:


ls -la

で実際のファイル名を確認します。

UnsupportedClassVersionError


UnsupportedClassVersionError

原因:

  • Java 21でビルド
  • Java 17で実行

対処:

DockerイメージのJavaバージョンを合わせます。

Port already in use


Port already in use

8080ポート競合です。

別ポートへ変更するか既存プロセスを停止します。

Communications link failure


Communications link failure

MySQL接続失敗です。

  • MySQL起動確認
  • ホスト名確認
  • サービス名確認

Application run failed


Application run failed

Spring Boot設定異常です。

スタックトレースの先頭付近に原因が出力されているためログを詳細確認してください。

Container exits immediately

コンテナ起動直後に停止する場合はENTRYPOINTやJar実行エラーの可能性があります。 


docker logs コンテナ名

で原因を確認します。

5. まとめ

  • まずdocker logsで原因を確認する
  • Jarファイル配置を確認する
  • Javaバージョンを合わせる
  • 8080ポート公開設定を確認する
  • MySQLやRedisなど依存サービスの起動状態を確認する
