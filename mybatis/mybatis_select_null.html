MyBatisでSELECT結果がnullになる原因を整理し、resultTypeとresultMapの不一致、カラム名ミスマッチなど実務で頻出する問題と解決方法を解説します。

1. 概要

MyBatisでSELECT結果がnullになる場合、多くはSQL自体ではなくマッピング設定の問題です。 特にresultTypeとresultMapの不一致や、DBカラム名とJavaプロパティ名のズレが原因になります。

項目 内容
環境 Java 21
フレームワーク Spring Boot 3.x
ORM MyBatis

2. 主な原因

2-1. resultTypeと実際のカラムが不一致

SQLのカラム名とJavaのプロパティ名が一致していないとnullになります。 


SELECT user_id AS userId, user_name AS userName
FROM users

2-2. resultMap未設定

複雑なマッピングの場合はresultMapを使わないと正しく格納されません。

2-3. データはあるがWHERE条件ミス

条件が厳しすぎて結果が0件→nullに見えるケースもあります。

SQLは正常でもマッピング失敗によりnullになることがあります。

5. まとめ

  • カラム名とJavaプロパティ名を一致させる
  • 必要に応じてresultMapを使う
  • WHERE条件も必ず確認する

SELECT結果がnullになる原因の多くは「マッピング設定ミス」です。まずはSQLとJavaの対応関係を確認してください。
