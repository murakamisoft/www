SQLが実行されない原因まとめ

MyBatisでSQLが「呼ばれているのに実行されない」ケースは、設定・トランザクション・ログ未確認の3つに大別できます。本記事では現場で頻出する原因を整理します。

1. 概要

「Mapperは呼ばれているのにDBに反映されない」「SELECT結果が空」などはMyBatis開発で非常に多い問題です。原因はSQLそのものより設定・実行環境側にあることがほとんどです。

項目 内容
対象 Spring Boot + MyBatis
典型症状 SQL未実行 / 更新されない / 結果が空

2. 主な原因

2-1. Mapperは呼ばれているがSQLログが出ない

SQLログが出ない場合、そもそもMyBatisがSQL実行層まで到達していません。

  • @MapperScan漏れ
  • Mapper未登録

2-2. XMLのnamespaceミス

namespaceがMapperインターフェースと一致していないとSQLは実行されません。 


<mapper namespace="com.example.mapper.UserMapper">

2-3. mapper-locationsのパスミス

XMLを読み込めていないケースです。


mybatis:
  mapper-locations: classpath:/mapper/*.xml

2-4. トランザクション未設定

@Transactionalがないと更新がロールバックされている可能性があります。

3. 切り分け方法

まず以下の順番で確認します。


① SQLログ出るか確認
② MapperがDIされているか確認
③ XMLが読み込まれているか確認
④ DB接続が正しいか確認

ログレベル設定：


logging:
  level:
    org.mybatis: debug
    java.sql: debug

4. よくあるエラー

① SQLが実行されない


MyBatisは呼ばれるがSQLログが出ない

→ Mapper未登録 or XML未読込

② 更新されない

→ トランザクション未設定（@Transactional不足）

③ SELECT結果が空

→ パラメータ不一致 or SQL条件ミス

5. まとめ

  • SQLが動かない原因はSQLではなく設定がほぼ原因
  • ログ確認で9割切り分け可能
  • Mapper・XML・TXの3点チェックが基本
