MyBatisとSpring Boot連携手順

Spring BootでMyBatisを使う場合、依存追加だけでは動かず「設定・スキャン・Mapper定義」の3点セットが必須です。本記事では最短で動かす構成を実務ベースで整理します。

1. 概要

MyBatisはSQLをXMLまたはアノテーションで管理できる軽量ORMです。Spring Bootと組み合わせる際は、設定漏れがあるとBean登録エラーやSQL未実行が発生します。

項目 内容
対象 Spring Boot 3.x + MyBatis
必要要素 依存関係 / @MapperScan / Mapper定義

2. 依存関係追加

pom.xmlにMyBatis Starterを追加します。


<dependency>
  <groupId>org.mybatis.spring.boot</groupId>
  <artifactId>mybatis-spring-boot-starter</artifactId>
  <version>3.0.3</version>
</dependency>

3. 設定ファイル

application.ymlにDB接続とMyBatis設定を追加します。


spring:
  datasource:
    url: jdbc:mysql://localhost:3306/testdb
    username: root
    password: root

mybatis:
  mapper-locations: classpath:/mapper/*.xml
  configuration:
    map-underscore-to-camel-case: true

@MapperScanを必ず設定します。


@SpringBootApplication
@MapperScan("com.example.mapper")
public class App {
  public static void main(String[] args) {
    SpringApplication.run(App.class, args);
  }
}

4. Mapper作成と動作確認

Mapperインターフェースを作成します。


@Mapper
public interface UserMapper {
    User findById(Long id);
}

対応するXML


<mapper namespace="com.example.mapper.UserMapper">

  <select id="findById" resultType="com.example.domain.User">
    SELECT id, name FROM user WHERE id = #{id}
  </select>

</mapper>

5. よくある失敗

① Bean登録されない

@MapperScan漏れが原因

② SQLが実行されない

mapper-locationsのパスミスが原因

③ MapperがDIできない

パッケージ構造がSpring Bootのスキャン外

  • @MapperScan確認
  • XMLパス確認
  • パッケージ構成確認

まとめ

  • MyBatisは設定依存が強い
  • 3点（依存・Scan・Mapper）で必ず動く
  • エラーの9割は設定ミス
