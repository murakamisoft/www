MyBatisのSQLログが出ない原因と解決方法

MyBatisでSQLログが出力されない原因を整理し、application.ymlやlogback設定ミス、debugログ不足など実務でよくある問題と対処法を解説します。

1. 概要

MyBatisではSQLログを確認することで、実行SQLの確認やバグ調査が可能です。 しかし設定ミスによりSQLログが出力されないケースがよくあります。

項目 内容
環境 Java 21
フレームワーク Spring Boot 3.x
ORM MyBatis

2. 主な原因

2-1. application.ymlのログレベル不足

debug設定がないとSQLログは出力されません。


logging:
  level:
    org.mybatis: DEBUG
    java.sql: DEBUG
    org.springframework.jdbc: DEBUG

2-2. logback設定不足

logback-spring.xmlでSQLログ出力設定が必要な場合があります。

2-3. MyBatis設定不足

configurationでlogImpl設定がないとログが出ないことがあります。


mybatis:
  configuration:
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl

3. サンプル設定

SQLログを有効化する標準設定例


logging:
  level:
    root: INFO
    org.mybatis: DEBUG

mybatis:
  configuration:
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl

4. よくあるエラー

SQLログが一切出ない


(no SQL log output)

原因はほぼログレベル設定不足かlogImpl未設定です。

5. まとめ

  • logging.levelをDEBUGにする
  • mybatis.log-implを設定する
  • logback設定も確認する

SQLログが出ない問題は設定ミスがほとんどです。まずはloggingとmybatis設定を確認してください。
