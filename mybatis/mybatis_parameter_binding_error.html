MyBatisのパラメータバインドエラー（#と$の違い）と安全な書き方

MyBatisの#{}と${}の違いを正しく理解しないと、SQLエラーやSQLインジェクションの原因になります。本記事では実務での安全な使い方とバインドエラーの対処法を解説します。

1. 概要

MyBatisではパラメータの埋め込み方法として「#{}」と「${}」の2種類があります。 この違いを理解していないと、SQLエラーやセキュリティ問題につながります。

項目 内容
環境 Java 21
フレームワーク Spring Boot 3.x
ORM MyBatis

2. #{}と${}の違い

2-1. #{}（推奨：安全）

PreparedStatementとしてバインドされるため、安全でSQLインジェクションを防げます。 


SELECT * FROM users WHERE id = #{id}

2-2. ${}（非推奨：危険）

文字列置換のためSQLインジェクションのリスクがあります。


SELECT * FROM users WHERE name = ${name}

基本的に${}はORDER BYやカラム名など限定用途のみ使用します。

バインドエラー例

原因はMapperメソッドとSQL側のパラメータ名不一致が多いです。

5. まとめ

  • #{}を基本的に使用する
  • ${}は限定用途のみ使用する
  • パラメータ名の不一致を確認する

MyBatisのバインドエラーは「書き方ルールの理解不足」が原因です。まずは#{}を徹底してください。
