MyBatisのINSERT/UPDATEが反映されない原因と解決方法

MyBatisでINSERTやUPDATEが実行されているのにDBに反映されない場合の原因を整理し、実務でよくあるトラブルと解決方法を解説します。

1. 概要

MyBatisではSQL自体は正常に実行されていても、トランザクション管理や設定ミスによってDBに反映されないケースがあります。 特にSpring Boot環境では自動コミットされないため注意が必要です。

項目 内容
環境 Java 21
フレームワーク Spring Boot 3.x
ORM MyBatis

2. 主な原因と対処法

2-1. トランザクション未設定

Springでは@Transactionalがないと、更新がコミットされず反映されないことがあります。 


@Transactional
public void updateUser(User user) {
    userMapper.update(user);
}

2-2. commit不足（非Spring管理の場合）

MyBatis単体利用の場合、commitを明示的に呼ばないと反映されません。


SqlSession session = sqlSessionFactory.openSession();
try {
    session.update("UserMapper.updateUser", user);
    session.commit();
} finally {
    session.close();
}

2-3. Mapper呼び出しは成功しているがロールバックされる

例外が発生すると自動的にロールバックされるため、ログ確認が重要です。

3. サンプルコード

Spring Boot + MyBatisの基本的なUPDATE処理例です。


@Mapper
public interface UserMapper {
    int updateUser(User user);
}

@Service
public class UserService {

    private final UserMapper userMapper;

    public UserService(UserMapper userMapper) {
        this.userMapper = userMapper;
    }

    @Transactional
    public void updateUserName(int id, String name) {
        User user = new User();
        user.setId(id);
        user.setName(name);

        userMapper.updateUser(user);
    }
}

4. よくあるエラー

例：更新してもDBが変わらない


SQL executed successfully but no changes in DB

原因はトランザクション未設定またはcommit漏れがほとんどです。

5. まとめ

  • @Transactionalの付け忘れを確認する
  • MyBatis単体利用時はcommit必須
  • 例外発生時のロールバックに注意

MyBatisの更新が反映されない問題の多くは設定ミスです。まずはトランザクション管理を重点的に確認してください。
