MyBatisのMapperが読み込まれない原因と解決方法

MyBatisでMapperインターフェースがDIされない・Beanとして認識されない原因を整理し、@MapperScan漏れやXML設定ミスなど実務で頻出する問題と解決方法を解説します。

1. 概要

MyBatisではMapperインターフェースがSpringのBeanとして登録されていないと、 DI失敗や起動エラーが発生します。特に設定漏れやパッケージ構成ミスが原因になることが多いです。

項目 内容
環境 Java 21
フレームワーク Spring Boot 3.x
ORM MyBatis

2. 主な原因

2-1. @MapperScanが設定されていない

Spring Bootでは@MapperScanを指定しないとMapperが自動登録されません。 


@SpringBootApplication
@MapperScan("com.example.mapper")
public class App {
}

2-2. @Mapperが付いていない

インターフェースに@Mapperがない場合もBeanとして認識されません。


@Mapper
public interface UserMapper {
    User findById(int id);
}

2-3. パッケージ構成ミス

Spring BootのComponentScan範囲外にMapperがあると読み込まれません。

3. サンプルコード

正しい構成例（推奨構成）


@SpringBootApplication
@MapperScan("com.example.mybatis.mapper")
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}

@Mapper
public interface UserMapper {
    User selectById(int id);
}

4. よくあるエラー

BeanCreationException


org.springframework.beans.factory.NoSuchBeanDefinitionException

MapperがBeanとして登録されていない場合に発生します。@MapperScanまたは@Mapperを確認してください。

5. まとめ

  • @MapperScanの設定漏れを確認する
  • @Mapperアノテーションを付ける
  • パッケージ構成をSpringのスキャン範囲内にする

Mapper関連エラーの多くは「設定漏れ」です。まずはスキャン設定を最優先で確認してください。
