MyBatisのN+1問題とパフォーマンス改善方法

MyBatisで発生するN+1問題の原因を整理し、不正なSELECTループの回避方法やJOIN最適化、SQLチューニングによるパフォーマンス改善方法を解説します。

1. 概要

N+1問題とは、1回の親クエリ取得後に、子データ取得のためにN回SQLが発行される問題です。 MyBatisでは特にループ内でのSELECT呼び出しにより発生しやすく、性能劣化の原因になります。

項目 内容
環境 Java 21
フレームワーク Spring Boot 3.x
ORM MyBatis

2. N+1問題の原因

2-1. ループ内でSELECTを実行している

親データ取得後に、子データを1件ずつSELECTするとN+1が発生します。 


for (User user : users) {
    List orders = orderMapper.findByUserId(user.getId());
}

2-2. Lazy Loadの誤用

遅延ロード設定により、アクセス時にSQLが発行されるケースがあります。

3. 改善方法

3-1. JOINで一括取得

JOINを使うことでSQL発行回数を1回に抑えられます。 


SELECT u.id, u.name, o.id, o.total
FROM users u
LEFT JOIN orders o ON u.id = o.user_id;

3-2. IN句でまとめて取得

複数IDをまとめて取得することでSQL回数を削減できます。 


<select id="findOrdersByUserIds">
  SELECT * FROM orders
  WHERE user_id IN
  <foreach collection="ids" item="id" open="(" close=")" separator=",">
    #{id}
  </foreach>
</select>

3-3. DTOでまとめてマッピング

JOIN結果をDTOにマッピングすることで効率的に取得できます。

4. サンプルコード


@Mapper
public interface UserMapper {

    List findUsersWithOrders();
}

SQL設計段階でJOIN前提にすることでN+1問題を防止できます。

5. まとめ

  • ループ内SELECTは避ける
  • JOINまたはIN句で一括取得する
  • DTO設計でデータ構造を最適化する

N+1問題はMyBatisの典型的な性能問題です。まずはSQL発行回数を意識することが重要です。
