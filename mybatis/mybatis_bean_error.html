MyBatisのBean登録エラー解決方法

Spring Boot + MyBatisで発生するBean登録エラーは、Mapper未検出・@MapperScan漏れ・DI設定ミスが原因のほぼ3パターンです。本記事では実務で頻出する原因と即解決方法を整理します。

1. 概要

Spring BootでMyBatisを使用している際、「Beanが見つからない」「MapperがDIできない」エラーは頻繁に発生します。原因の多くは設定不足です。

項目 内容
対象 Spring Boot + MyBatis
主な原因 @MapperScan不足 / Mapper未登録 / パッケージ不一致

2. 主な原因と解決手順

2-1. @MapperScanの設定漏れ

MapperインターフェースをSpringが認識していないケースです。必ず起動クラスに指定します。 


@SpringBootApplication
@MapperScan("com.example.mapper")
public class App {
  public static void main(String[] args) {
    SpringApplication.run(App.class, args);
  }
}

2-2. Mapperにアノテーション不足

@Mapperを付けない場合、Bean登録されずDIできません。


@Mapper
public interface UserMapper {
    User findById(Long id);
}

2-3. コンポーネントスキャン範囲外

パッケージ構成がずれているとBeanが見つかりません。SpringBootApplicationの階層に注意します。

3. 正しい設定例

最小構成の正しい例は以下です。


@SpringBootApplication
@MapperScan("com.example.mapper")
public class App {
}

@Mapper
public interface UserMapper {
    User findById(Long id);
}

4. よくあるエラーケース

エラー例


NoSuchBeanDefinitionException: No qualifying bean of type 'xxxMapper'

このエラーはほぼ「MapperがSpringに登録されていない」ことが原因です。

チェックリスト

  • @MapperScanが設定されているか
  • @Mapperが付いているか
  • パッケージ構成が起動クラス配下か
  • build.gradle / pom.xmlにmybatis依存があるか

5. まとめ

  • Beanエラーの原因はほぼ設定ミス
  • @MapperScanと@Mapperが最重要
  • パッケージ設計が崩れると再発する
