MyBatisで動的SQLが動かない原因（if / where / foreach）と解決方法

MyBatisの動的SQLが意図通り動かない原因を整理し、if / where / foreachタグのミスやOGNL条件エラーなど実務で頻出する問題と対処法を解説します。

1. 概要

MyBatisの動的SQLは便利ですが、条件式の書き方やタグの構文ミスで「SQLが生成されない」「条件が効かない」などの問題が頻発します。

項目 内容
環境 Java 21
フレームワーク Spring Boot 3.x
ORM MyBatis

2. よくある原因

2-1. OGNL条件ミス（if文が常にfalse）

nullチェックや文字列比較の書き方ミスでSQLが生成されないケースがあります。

2-2. whereタグの使い方ミス

whereタグを使わないとSQLの先頭AND/ORが原因で構文エラーになります。

2-3. foreachの構文ミス

collection名やitem指定ミスでSQLが生成されません。 



  #{id}

3. サンプルコード

条件付き検索の基本例です。

4. よくあるエラー

SQLが生成されない


org.apache.ibatis.exceptions.PersistenceException

原因はif条件がfalse、またはOGNL式ミスの可能性が高いです。

5. まとめ

  • OGNL条件式のミスを疑う
  • whereタグでAND問題を回避する
  • foreachのcollection指定を確認する

動的SQLのトラブルはほぼ「構文ミス」か「条件式ミス」です。まずはログを確認してSQL生成状況をチェックしてください。
