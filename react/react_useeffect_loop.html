useEffectが無限ループする原因と対処法

Reactで頻出する「無限再レンダリング」の原因と、正しいuseEffect設計を解説

最終更新日：2026-06-13

目次

  • 1. 概要
  • 2. 無限ループの原因
  • 3. 正しい実装方法
  • 4. よくあるミス
  • 5. まとめ

1. 概要

useEffectは副作用処理（API通信・イベント登録など）を行うためのHookですが、依存配列の設計ミスにより無限ループが発生しやすいポイントです。

2. 無限ループの原因

典型例は「useEffect内でstate更新し、そのstateを依存配列に入れているケース」です。 

 useEffect(() => { setCount(count + 1); }, [count]);

この場合、count更新 → useEffect再実行 → count更新…の循環が発生します。

3. 正しい実装方法

依存関係を適切に分離し、更新トリガーを制御します。

 useEffect(() => { setCount(prev => prev + 1); }, []);

または、状態更新が不要な場合はuseEffect内でstate変更を行わない設計にします。

4. よくあるミス

  • 依存配列に不要なstateを入れる
  • オブジェクト・配列を直接依存にする
  • useEffect内で常にsetStateする

5. まとめ

  • state更新とuseEffect依存関係の循環に注意
  • 依存配列は最小限にする
  • 関数型更新を活用する
