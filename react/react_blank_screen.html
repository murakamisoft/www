Reactで画面が真っ白になる原因まとめ（完全デバッグガイド）

Reactで画面が真っ白になる現象は、ビルド失敗・JavaScriptエラー・ルーティングミス・レンダリング例外など複数原因が絡みます。本記事では現場で即特定できる切り分け手順を整理します。

1. 概要

Reactの「画面が真っ白になる」問題は、エラーが画面に表示されないケースも多く、初心者が最も詰まりやすいポイントです。 実務では「コンソール確認」と「レンダリング失敗の切り分け」が最重要になります。

項目 内容
対象 React / Next.js
症状 画面が完全に白い / コンポーネントが表示されない
原因領域 JSエラー / ルーティング / ビルド / state破損

2. 主な原因と切り分け

2-1. JavaScriptエラー（最優先確認）

consoleにエラーが出ている場合、レンダリングは完全停止します。 



Uncaught TypeError: Cannot read property ...

対処：該当コンポーネントを特定してnullチェックを追加

2-2. ルーティングミス

React Routerでpathが一致しない場合、何も表示されないことがあります。 



} />

URLとRoute定義の不一致を確認。

2-3. ビルド失敗・importエラー

importパスミスやコンパイルエラーでも画面は真っ白になります。 



Module not found: Can't resolve './components'

4. よくあるエラーと実務対応

① blank screen + console errorなし

ルーティング or return nullの可能性。

② useEffect内エラーで全停止

非同期処理の例外が未catch。

③ state初期値ミス

undefinedアクセスでレンダリング崩壊。



const [data, setData] = useState(null);
console.log(data.name); // NG

④ CSSで非表示

display:noneやopacityで見えていないだけのケース。

5. まとめ

  • 真っ白はJSエラーが最優先原因
  • ルーティングミスは頻出
  • stateのundefinedアクセスは致命的
  • 最小構成で切り分けるのが最短ルート

Reactのデバッグは「原因特定の手順化」ができると一気に速くなります。感覚ではなく切り分けフローで対応するのが重要です。
