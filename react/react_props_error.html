propsが渡らない原因と対処法（コンポーネント設計ミス完全解決）

Reactで最も多いバグの1つが「propsが渡っていないように見える問題」です。本記事では名前ミス・分割代入・レンダリング構造など、現場で頻出する原因を体系的に整理します。

1. 概要

propsは親コンポーネントから子コンポーネントへデータを渡す基本機能ですが、 名前の不一致や分割代入ミスにより「渡っていないように見える」ケースが多発します。

項目 内容
対象 React
問題領域 コンポーネント設計 / データ受け渡し
主な原因 名前ミス / 分割代入 / 再レンダリング / undefined

2. 主な原因

2-1. props名のミス

親と子で名前が一致していないケース。





// 子
function Child(props) {
console.log(props.username); // NG（小文字違い）
}

2-2. 分割代入ミス

propsの取り出し方が間違っているケース。



function Child({ userName }) {
return 
{userName}
;
}

2-3. 親から渡していない

単純にprops未指定。



 // userName未設定

3. 正しい実装例

基本形：



function Parent() {
return ;
}

function Child({ userName }) {
return 
{userName}
;
}

デバッグ方法：



console.log(props);

4. よくあるエラー

① undefinedが表示される

props未定義または名前ミス。

② 値が更新されない

stateとpropsの混同。

③ 子コンポーネントが再レンダリングされない

memo化の影響。

5. まとめ

  • propsは名前一致が絶対条件
  • 分割代入ミスが頻出
  • 親の渡し忘れも多い
  • console.logで即確認するのが最速

props問題は「構造確認」でほぼ解決できます。バグというより設計チェックの問題です。
