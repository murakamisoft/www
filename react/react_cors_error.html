1. 概要

CORS（Cross-Origin Resource Sharing）は、異なるドメイン間の通信を制御するブラウザのセキュリティ機構です。 ReactからAPIを呼び出す際に最もよく発生するエラーです。

項目 内容
対象 React / Vue / Angular
発生場所 ブラウザ（クライアント側）
原因 異なるオリジン間通信の制限

2. CORSの仕組み

ブラウザは「同一オリジンポリシー」により、異なるドメインのAPI通信をデフォルトで禁止しています。 



例：
フロント: http://localhost:3000
API : http://localhost:8080
→ 別オリジン扱い

この制限を解除するにはサーバー側で明示的に許可が必要です。

3. 対処方法

3-1. サーバー側でCORS許可（推奨）



@CrossOrigin(origins = "http://localhost:3000")
@RestController
public class ApiController {
}

3-2. Spring Boot設定



@Configuration
public class CorsConfig {

@Bean
public WebMvcConfigurer corsConfigurer() {
return new WebMvcConfigurer() {
@Override
public void addCorsMappings(CorsRegistry registry) {
registry.addMapping("/**")
.allowedOrigins("")
.allowedMethods("");
}
};
}
}

3-3. フロント側Proxy（開発用）



"proxy": "http://localhost:8080"

4. よくあるエラーと落とし穴

① Preflight（OPTIONS）エラー

PUT/DELETEなどで事前リクエストが失敗するケース。

② credentialsエラー

cookieを使う場合に発生。

③ allow-origin '*' 問題

認証付き通信では使用不可。

④ フロント側で解決しようとする誤り

基本的にCORSはサーバー側の責任。

5. まとめ

  • CORSはブラウザのセキュリティ制御
  • 基本はサーバー側で解決する
  • 開発中はproxyで回避可能
  • 本番では正しいオリジン設定が必須

CORSは「フロントで直す問題ではない」という理解が最重要です。
