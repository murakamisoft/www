stateが更新されない原因と解決方法（useState完全理解）

Reactのstateが更新されない問題は「仕様の誤解」が原因のことが多いです。本記事ではuseStateの挙動・非同期更新・参照ミスなどを体系的に整理します。

1. 概要

Reactのstateは「即時更新される変数」ではなく「再レンダリングをトリガーする値」です。 この仕様を理解していないと、更新されないように見える問題が発生します。

項目 内容
対象 React / useState
問題領域 状態管理
主な原因 非同期更新 / 直接代入 / 参照ミス

2. 主な原因

2-1. 非同期更新

setState後すぐに値を読んでいるケース。



setCount(count + 1);
console.log(count); // 更新前の値

2-2. 直接代入

stateを直接書き換えている。



count = 10; // NG

2-3. 参照型のミス

配列やオブジェクトの直接変更。



state.push(1); // NG

3. 正しい使い方



import { useState } from "react";

function Counter() {
const [count, setCount] = useState(0);

const increment = () => {
setCount(prev => prev + 1);
};

return (

);
}

ポイント：必ずsetStateを使うこと。

4. よくあるエラー

① stateが変わらない

直接代入が原因。

② 最新値が取れない

非同期更新の理解不足。

③ 配列が更新されない

ミューテーション（直接変更）。

5. まとめ

  • stateは即時反映されない
  • 直接代入は禁止
  • 参照型はコピーして更新
  • setStateで必ず更新する

useStateは「変数」ではなく「レンダリングトリガー」という理解が重要です。
