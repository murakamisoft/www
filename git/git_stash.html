git stashの使い方とよくある失敗例

「別のブランチで作業しなきゃいけないのに、今の作業をコミットしたくない…」。そんな時に便利なgit stashの使い方を解説します。作業を一時的に隠して、安全に別の作業へ切り替える方法を学びましょう。

1. 概要：stashとは何か

git stashは、まだコミットしていない作業中の変更（ファイルの状態）を、一時的に専用のスタック領域へ退避させるコマンドです。

項目 内容
主な用途 ブランチ切り替え、緊急修正のための作業一時停止
メリット コミットせずに作業をクリーンな状態にできる

2. 基本手順：退避と復元

2-1. 作業の退避


# 作業内容を一時退避
git stash

# 退避したリストを確認
git stash list

2-2. 退避内容の取り出し


# 最新のスタッシュを取り出してリストから削除
git stash pop

3. よくある失敗と注意点

スタッシュでよくある失敗は、popした際にコンフリクトが発生することです。

退避してから時間が経ち、その間に他のメンバーが同じ箇所を修正してプッシュしていた場合、pop時に競合が発生します。その場合は、マージコンフリクトと同様に手動で修正する必要があります。

注意点: 新規作成したファイル（Untracked files）は、デフォルトではgit stashに含まれません。含めるには git stash -u を使用してください。

Adsense Ad Unit

4. 応用コマンド


# 名前を付けて保存（識別しやすくなる）
git stash save "作業のメモ"

# スタッシュの中身を破棄する
git stash drop

5. まとめ

  • git stashは作業中の中途半端な状態を一時避難させる機能。
  • popで戻せるが、コンフリクトする可能性がある。
  • 新規ファイルは -u オプションを忘れないようにしよう。
