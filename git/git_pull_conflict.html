git pullでコンフリクトが発生する原因と解決方法

チーム開発で頻繁に発生する「git pull時のコンフリクト」。リモートリポジトリの変更と自分のローカル環境での変更が重なったとき、どのように対処すれば安全に最新の状態へ同期できるのかを解説します。

1. 概要：なぜpullで衝突するのか

git pullは「リモートの変更を取得（fetch）」し、それを「現在のブランチに統合（merge）」する操作です。ローカルで行った編集と、他のメンバーがリモートにプッシュした内容が同じファイル・同じ場所で重なった場合、Gitは自動統合できず「コンフリクト（競合）」を発生させます。

項目 内容
発生原因 ローカルとリモート双方での同一ファイル編集
重要性 放置するとチーム開発の進捗に影響が出る

2. 手順：コンフリクト解決のフロー

2-1. コンフリクトの確認

git pullを実行してエラーが出たら、まずは状況を確認します。 


git status
# 「both modified: ファイル名」と表示される

2-2. 解決の手順

エディタで競合箇所を確認し、不要なマーク（<<<<< 等）を削除してコードを修正します。 


# 修正が終わったらステージングに追加
git add .

# マージコミットを作成
git commit -m "Resolve pull conflict"

3. 実践：安全なマージ処理

誤ったマージを避けるため、作業前に一度stashで退避させる方法も有効です。 


# 変更を一時退避
git stash

# 安全に最新を取得
git pull

# 退避した内容を戻す
git stash pop

4. よくあるトラブル

「マージが中途半端で戻したい」場合


# pullで始まったマージ処理をキャンセルする
git merge --abort

解決できないまま進めてしまうと、コミット履歴が複雑になります。一度リセットして落ち着いて対処しましょう。

5. まとめ

  • git pull中のコンフリクトはチーム開発では避けて通れない。
  • git statusで場所を確認し、手動修正後にコミットする。
  • 不安な場合はgit stashを利用して安全にマージする。
