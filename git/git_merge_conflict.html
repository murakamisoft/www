merge conflictの解決方法まとめ

Gitでブランチをマージしようとした際、「CONFLICT (content): Merge conflict in...」と表示されて処理が止まってしまったことはありませんか？本記事では、コンフリクトの発生原因と、安全に解決するための手順を解説します。

1. 概要：なぜコンフリクトが起きるのか

マージコンフリクトは、異なるブランチで同じファイルの同じ行がそれぞれ変更され、Gitがどちらの変更を優先すべきか自動判断できない時に発生します。

項目 内容
発生条件 異なるブランチで同一箇所の変更が競合した場合
Gitの挙動 自動マージを停止し、ユーザーに手動での修正を求める

2. 手順：コンフリクト解決のステップ

2-1. 競合しているファイルを確認する

まずは以下のコマンドで、どのファイルが競合しているかを特定します。 


git status
# 「both modified: ファイル名」と表示される箇所が競合中

2-2. ファイルを修正してコミットする

ファイル内のマークを参考に修正し、再度ステージングしてコミットします。 


# 修正後にステージング
git add ファイル名

# マージのコミットを完了させる
git commit -m "Fix merge conflict"

3. 実践：競合箇所の修正例

競合が発生したファイルを開くと、以下のような記号が含まれています。これらを削除し、意図するコードに書き換えます。 


<<<<<<< HEAD
現在のブランチのコード
=======
マージしようとしているブランチのコード
>>>>>>> branch-name

修正のポイント: 上記の <<<<<<< から >>>>>>> までをすべて削除し、正しい状態のコードに編集してください。

4. よくあるエラーとQ&A

Q: コンフリクトが複雑すぎて元に戻したい


# マージを開始する前の状態にリセットする
git merge --abort

解決が困難だと感じた場合は、一度--abortコマンドでマージ自体を中止し、状況を整理することをおすすめします。

5. まとめ

  • git statusで競合ファイルを特定する。
  • ファイル内の記号を削除し、正しいコードを記述する。
  • git add と git commit でマージを完了させる。
