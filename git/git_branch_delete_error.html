Gitでブランチ削除できない原因と対処法

Gitで不要なブランチを削除しようとした際に発生するエラーの原因と解決方法を解説します。 ローカルブランチ・リモートブランチの両方の削除方法を理解しましょう。

1. ブランチ削除エラーの概要

Gitでは誤操作によるデータ消失を防ぐため、安全でない状態のブランチ削除を禁止しています。 そのため、単純に git branch -d を実行しても削除できないケースがあります。

代表的な原因は次の通りです。

原因 説明
現在のブランチを削除している チェックアウト中のブランチは削除できない
未マージの変更が残っている Gitが安全のため削除を拒否する
リモートブランチを削除している 削除方法が異なる
権限不足 GitHubやGitLabの保護ブランチ設定

2. ブランチを削除する方法

2-1. 現在のブランチを確認する

まず削除対象ブランチにいる場合は別ブランチへ切り替える必要があります。 



git branch

* が付いているブランチが現在作業中のブランチです。



git switch main

2-2. ブランチを削除する

マージ済みブランチは -d オプションで削除できます。



git branch -d feature/sample

未マージでも強制的に削除する場合は -D を使用します。



git branch -D feature/sample

3. よく使う削除コマンド

実務でよく利用するブランチ削除コマンドをまとめます。


ローカルブランチ削除

git branch -d feature/sample

強制削除

git branch -D feature/sample

リモートブランチ削除

git push origin --delete feature/sample

リモート情報更新

git fetch --prune

開発チームでは feature ブランチをマージ後に削除する運用が一般的です。 定期的に不要ブランチを整理すると管理しやすくなります。

4. よくあるエラーと対処法

エラー1：現在のブランチを削除している



error: Cannot delete branch 'feature/sample'
checked out at ...

原因は削除対象ブランチにチェックアウトしていることです。 mainやdevelopへ切り替えてから削除してください。 



git switch main
git branch -d feature/sample

エラー2：未マージの変更がある



error: The branch 'feature/sample'
is not fully merged.

ブランチのコミットがまだマージされていません。 本当に不要なら -D で削除できます。 



git branch -D feature/sample

エラー3：リモートブランチが削除できない



remote: error
failed to push some refs

GitHubやGitLabで保護ブランチ（Protected Branch）が設定されている可能性があります。 mainやmasterは削除禁止になっていることがあります。

エラー4：削除したのに一覧に残る



git branch -a

リモート追跡情報が残っている場合があります。 次のコマンドで不要な参照を削除できます。 



git fetch --prune

5. まとめ

  • 現在のブランチは削除できない
  • マージ済みなら git branch -d を使用する
  • 未マージなら git branch -D で強制削除できる
  • リモートブランチは git push origin --delete を使用する
  • 不要な追跡情報は git fetch --prune で整理する
