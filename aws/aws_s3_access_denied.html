S3のAccess Deniedエラーの原因と対処法

AWS S3でAccess Deniedエラーが発生した場合の調査方法と解決手順を解説します。

1. Access Deniedが発生する主な原因

S3のAccess Deniedエラーは権限設定の問題で発生します。

原因 内容
IAMポリシー不足 s3:GetObjectやs3:PutObjectが許可されていない
Bucket Policy バケット側でアクセス拒否している
Public Access Block パブリックアクセスがブロックされている
ACL設定 オブジェクト権限不足
KMS権限不足 暗号化キーへのアクセス権がない
別アカウントアクセス クロスアカウント権限不足

2. 調査手順

2-1. IAMユーザーを確認する

まず現在利用している認証情報を確認します。


aws sts get-caller-identity

想定したIAMユーザーまたはIAMロールになっているか確認してください。

2-2. S3アクセス確認


aws s3 ls
        

aws s3 ls s3://sample-bucket

この時点でAccess Deniedが発生する場合は権限設定に問題があります。

2-3. Bucket Policy確認


aws s3api get-bucket-policy \
--bucket sample-bucket

明示的なDeny設定が存在するとAllow設定より優先されます。

2-4. Public Access Block確認

AWSではデフォルトでパブリックアクセスをブロックしています。

Web公開用途の場合は設定を確認してください。

3. IAMポリシー設定例

オブジェクト読み書きを許可するサンプルです。


{
  "Version":"2012-10-17",
  "Statement":[
    {
      "Effect":"Allow",
      "Action":[
        "s3:GetObject",
        "s3:PutObject",
        "s3:DeleteObject"
      ],
      "Resource":[
        "arn:aws:s3:::sample-bucket/*"
      ]
    }
  ]
}

バケット一覧取得にはs3:ListBucketも必要です。


{
  "Effect":"Allow",
  "Action":"s3:ListBucket",
  "Resource":"arn:aws:s3:::sample-bucket"
}

4. よくあるエラー

AccessDenied


An error occurred
(AccessDenied)
when calling the GetObject operation

GetObject権限不足です。

403 Forbidden


403 Forbidden

Bucket PolicyまたはPublic Access Block設定を確認してください。

Access Denied when calling PutObject


Access Denied
when calling PutObject

PutObject権限が不足しています。

KMS Access Denied


AccessDeniedException
kms:Decrypt

KMSキーへのアクセス権が不足しています。

5. まとめ

  • IAMポリシーを確認する
  • Bucket Policyを確認する
  • Public Access Blockを確認する
  • ACL設定を確認する
  • KMS権限を確認する

S3のAccess DeniedはIAMポリシーだけでなくBucket PolicyやPublic Access Blockも関係します。権限周りを順番に確認することで原因を特定できます。
