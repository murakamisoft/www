CloudWatchログが出ない原因と解決方法

AWS CloudWatch Logsにログが出力されない場合の原因と対処法を、EC2・Lambda・CloudWatch Agentのパターン別に解説します。

1. CloudWatchログが出ない主な原因

CloudWatch Logsへログが出力されない場合、多くはIAM権限不足やCloudWatch Agent設定ミスが原因です。

原因 発生頻度 確認ポイント
IAM権限不足 高 CloudWatchLogs権限
Log Group未作成 高 CloudWatchコンソール
Agent未起動 高 サービス状態確認
設定ファイル誤り 中 agent-config.json
リージョン違い 中 AWS Region確認

2. 確認すべき設定項目

2-1. IAMロール確認

EC2やECSからCloudWatch Logsへ出力する場合、IAMロールに以下の権限が必要です。 


{
  "Effect": "Allow",
  "Action": [
    "logs:CreateLogGroup",
    "logs:CreateLogStream",
    "logs:PutLogEvents"
  ],
  "Resource": "*"
}

2-2. Log Group確認

CloudWatchコンソールからLog Groupが存在するか確認します。

  • CloudWatch
  • Logs
  • Log groups

指定したロググループが存在しない場合は作成します。

2-3. CloudWatch Agent状態確認

LinuxサーバーではAgentが起動しているか確認します。


sudo systemctl status amazon-cloudwatch-agent

停止している場合は起動します。


sudo systemctl start amazon-cloudwatch-agent

3. CloudWatch Agent設定例

Spring BootログをCloudWatchへ転送する設定例です。


{
  "logs": {
    "logs_collected": {
      "files": {
        "collect_list": [
          {
            "file_path": "/var/log/application.log",
            "log_group_name": "springboot-prod",
            "log_stream_name": "{instance_id}"
          }
        ]
      }
    }
  }
}

設定反映後にAgentを再起動します。


sudo systemctl restart amazon-cloudwatch-agent

4. よくあるエラーと対処法

AccessDeniedException


AccessDeniedException

原因

  • IAM権限不足

対処法

  • CloudWatch Logs権限追加
  • IAMロール再アタッチ

Log stream does not exist


The specified log stream does not exist

原因

  • Log Stream未作成
  • ロググループ名誤り

対処法

  • Log Group確認
  • Agent設定確認

ログが1件も表示されない

原因

  • Agent停止
  • 監視対象ファイル誤り
  • リージョン違い

対処法


sudo systemctl status amazon-cloudwatch-agent
        

aws configure get region

5. まとめ

  • CloudWatchログ未出力の多くはIAM権限不足
  • CloudWatch Agentの起動状態を確認する
  • Log Group名とLog Stream名を確認する
  • リージョン設定の不一致に注意する
  • Agent設定変更後は再起動する
