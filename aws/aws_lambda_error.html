AWS Lambdaが実行されない原因と対処法まとめ

Lambdaが起動しない、トリガーが反応しない、CloudWatch Logsに何も出力されない場合の調査方法を解説します。

1. Lambdaが実行されない主な原因

Lambdaが実行されない場合、コード自体ではなく設定ミスが原因であるケースが非常に多くあります。

原因 内容
IAM権限不足 Lambda実行ロールに必要な権限が付与されていない
トリガー設定ミス S3、API Gateway、EventBridgeなどとの連携不備
CloudWatch Logs権限不足 ログ出力できず原因調査が困難になる
タイムアウト 処理時間が制限時間を超過している
環境変数ミス 接続先URLや認証情報が誤っている
リージョン違い Lambdaとイベント発生元が別リージョン

2. 調査手順

2-1. CloudWatch Logsを確認する

Lambda障害調査で最初に確認すべきなのがCloudWatch Logsです。 


aws logs describe-log-groups

aws logs tail \
/aws/lambda/sample-function \
--follow

エラーメッセージが出力されている場合は、まずその内容を確認しましょう。

2-2. Lambda実行ロールを確認する

LambdaはIAMロールの権限でAWSサービスへアクセスします。

最低限以下のポリシーが必要になることが多いです。


{
  "Version":"2012-10-17",
  "Statement":[
    {
      "Effect":"Allow",
      "Action":[
        "logs:CreateLogGroup",
        "logs:CreateLogStream",
        "logs:PutLogEvents"
      ],
      "Resource":"*"
    }
  ]
}

2-3. トリガー設定を確認する

Lambda関数単体では実行されません。イベントソースとの連携が必要です。

  • S3イベント
  • API Gateway
  • EventBridge
  • SQS
  • SNS
  • DynamoDB Streams

トリガー画面で対象リソースが正しく設定されているか確認してください。

3. Lambda実装例

Java Lambdaの基本実装例です。 


package sample;

import com.amazonaws.services.lambda.runtime.Context;
import com.amazonaws.services.lambda.runtime.RequestHandler;

public class HelloLambda
implements RequestHandler {

    @Override
    public String handleRequest(
        String input,
        Context context) {

        context.getLogger()
               .log("input=" + input);

        return "Hello Lambda";
    }
}

まずはログ出力だけの簡単な実装で動作確認すると原因を切り分けやすくなります。

4. よくあるエラーと対処法

AccessDeniedException


AccessDeniedException:
User is not authorized
to perform this action

IAMロールの権限不足です。必要なポリシーを追加してください。

Task timed out


Task timed out
after 3.00 seconds

Lambdaのタイムアウト設定が短すぎる可能性があります。

  • タイムアウト値を延長する
  • 外部API呼び出し時間を短縮する
  • 処理を非同期化する

CloudWatch Logsに何も出ない

Lambdaが起動していないか、ログ出力権限が不足している可能性があります。

  • トリガー設定確認
  • IAMロール確認
  • リージョン確認

S3アップロードで起動しない

S3イベント通知が未設定であるケースが多くあります。

  • Bucket Notification設定確認
  • 対象プレフィックス確認
  • 対象サフィックス確認

5. まとめ

  • まずCloudWatch Logsを確認する
  • IAMロールの権限不足を疑う
  • トリガー設定を確認する
  • リージョン違いを確認する
  • タイムアウトや環境変数もチェックする

Lambda障害の大半は「IAM」「トリガー」「CloudWatch Logs」の3点を確認することで原因を特定できます。
