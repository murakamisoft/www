EC2でポートが開かない原因（セキュリティグループ）

EC2にデプロイしたアプリへアクセスできない場合の原因と対処法を解説します。Spring Boot、Nginx、Apacheで発生する通信エラーの切り分け手順も紹介します。

1. ポートが開かない主な原因

EC2で通信できない場合、原因の約8割はセキュリティグループ設定ミスです。

原因 発生頻度 確認場所
セキュリティグループ未設定 非常に高い AWSコンソール
OSファイアウォール 高い Linux設定
アプリ未起動 高い プロセス確認
Listen Address設定ミス 中 Spring Boot設定
NACL設定ミス 低い VPC設定

2. 確認手順

2-1. セキュリティグループ確認

まずはインバウンドルールを確認します。

タイプ ポート 送信元
HTTP 80 0.0.0.0/0
HTTPS 443 0.0.0.0/0
Spring Boot 8080 0.0.0.0/0

2-2. アプリ起動確認

Spring Bootが起動しているか確認します。


ps -ef | grep java

2-3. ポート監視確認


sudo netstat -tulpn

または以下でも確認できます。


sudo ss -tulpn

3. 実際の確認コマンド

ポート8080確認


sudo ss -tulpn | grep 8080

正常例


LISTEN 0 100 *:8080 *:*

ローカル接続確認


curl localhost:8080

外部接続確認


curl http://EC2_PUBLIC_IP:8080

Spring Boot設定確認


server.port=8080
server.address=0.0.0.0

localhost指定になっていると外部アクセスできません。

4. よくあるエラー

Connection timed out


Connection timed out

原因

  • セキュリティグループ未設定
  • NACL拒否

対処法

  • インバウンドルール確認
  • VPC設定確認

Connection refused


Connection refused

原因

  • アプリ未起動
  • ポート監視なし

対処法

  • プロセス確認
  • ログ確認

ERR_CONNECTION_TIMED_OUT

原因

  • 通信遮断
  • セキュリティグループ設定漏れ

対処法

  • 80/443ポート開放
  • EC2再起動後に再確認

5. まとめ

  • まずセキュリティグループを確認する
  • アプリが起動しているか確認する
  • ポート監視状態を確認する
  • server.address=0.0.0.0 を確認する
  • 切り分けは「AWS → OS → アプリ」の順で行う
