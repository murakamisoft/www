EC2にSSH接続できない原因と解決方法

AWS EC2へSSH接続できない場合の原因と対処法を解説します。Security Group設定、秘密鍵エラー、ユーザー名間違いなどの定番トラブルをまとめています。

1. SSH接続できない主な原因

EC2のSSH接続エラーは、Security Group設定または秘密鍵設定の問題が大半を占めます。

原因 発生頻度 確認箇所
Security Group未設定 非常に高い AWSコンソール
秘密鍵間違い 高い .pemファイル
ユーザー名間違い 高い SSHコマンド
パブリックIP未設定 中 EC2設定
Route Table設定ミス 低い VPC設定

2. 確認手順

2-1. Security Group確認

SSH用の22番ポートが開放されていることを確認します。

タイプ ポート 送信元
SSH 22 自分のIPアドレス

本番環境では 0.0.0.0/0 ではなく、自分のIPアドレスのみ許可することを推奨します。

2-2. パブリックIP確認

EC2インスタンスにパブリックIPが割り当てられているか確認します。

2-3. インスタンス状態確認

EC2が起動状態（Running）になっていることを確認します。

2-4. ユーザー名確認

AMI ユーザー名
Amazon Linux ec2-user
Ubuntu ubuntu
CentOS centos
Debian admin

3. SSH接続例

Amazon Linuxへ接続する例です。


ssh -i mykey.pem ec2-user@54.xxx.xxx.xxx

秘密鍵権限変更


chmod 400 mykey.pem

疎通確認


telnet 54.xxx.xxx.xxx 22

または以下のコマンドでも確認できます。


nc -vz 54.xxx.xxx.xxx 22

4. よくあるエラー

Permission denied (publickey)


Permission denied (publickey)

原因

  • 秘密鍵間違い
  • ユーザー名間違い

対処法

  • pemファイル確認
  • AMIのユーザー名確認

Connection timed out


Connection timed out

原因

  • 22番ポート未開放
  • パブリックIPなし

対処法

  • Security Group確認
  • VPC設定確認

Connection refused


Connection refused

原因

  • SSHサービス停止

対処法

  • EC2 Instance Connectでログイン
  • sshdサービス確認

sudo systemctl status sshd

UNPROTECTED PRIVATE KEY FILE


WARNING: UNPROTECTED PRIVATE KEY FILE!

原因

  • 秘密鍵の権限が広すぎる

対処法


chmod 400 mykey.pem

5. まとめ

  • まずSecurity Groupの22番ポートを確認する
  • パブリックIPの有無を確認する
  • AMIごとのユーザー名を確認する
  • pemファイルの権限を確認する
  • エラー内容から原因を切り分ける
