1. AWS CLIとは

AWS CLI（Command Line Interface）は、コマンドラインからAWSサービスを操作するための公式ツールです。 EC2、S3、RDS、Lambdaなど、多くのAWSサービスをブラウザを開かずに操作できます。

AWSの学習や運用を始めると最初に利用するツールですが、認証情報やIAM権限の設定ミスによってエラーになるケースが非常に多くあります。

利用環境 Windows / Linux / macOS

2. AWS CLIが動かない主な原因

2-1. AWS CLIがインストールされていない

まずはCLIが正しくインストールされているか確認します。


aws --version

正常な場合は以下のような結果が表示されます。


aws-cli/2.x.x Python/3.x Windows/10 exe/AMD64

2-2. 認証情報が設定されていない

AWS CLIで最も多い原因です。


aws configure

IAMユーザーのアクセスキーとシークレットキーを設定します。

2-3. リージョン設定が誤っている

東京リージョンを利用する場合は ap-northeast-1 を指定します。


Default region name: ap-northeast-1

2-4. IAM権限不足

認証は成功していてもIAMポリシーに必要な権限がないと操作できません。

3. AWS CLIの設定手順

手順1：アクセスキーを作成

AWSマネジメントコンソールからIAMユーザーを作成し、アクセスキーを発行します。

手順2：aws configureを実行


aws configure

手順3：認証確認


aws sts get-caller-identity

4. よくあるエラーと対処法

Unable to locate credentials


Unable to locate credentials

原因

  • aws configure未実施
  • 認証情報ファイル削除

対処法


aws configure

AccessDenied


An error occurred (AccessDenied)

原因

  • IAM権限不足
  • ロール設定ミス

対処法

  • IAMポリシーを確認する
  • 必要な権限を追加する

InvalidClientTokenId


The security token included in the request is invalid

原因

  • アクセスキー誤入力
  • アクセスキー失効

対処法

  • アクセスキー再発行
  • aws configure再実行

Could not connect to the endpoint URL


Could not connect to the endpoint URL

原因

  • リージョン設定ミス
  • プロキシ設定ミス
  • ネットワーク障害

対処法


aws configure get region

5. まとめ

  • AWS CLIエラーの多くは認証設定ミスが原因
  • まず aws --version を確認する
  • aws configureで認証情報を設定する
  • IAM権限不足も頻出原因の一つ
  • aws sts get-caller-identityで疎通確認できる
