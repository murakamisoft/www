RDSに接続できない原因と解決方法

Spring BootやJavaアプリケーションからAWS RDSへ接続できない場合の確認ポイントを解説します。

1. RDS接続エラーの主な原因

RDS接続エラーの多くはネットワーク設定または認証情報の設定ミスによって発生します。

原因 内容
Security Group 接続元IPやEC2が許可されていない
ユーザー名・パスワード 認証情報が間違っている
JDBC URL ホスト名やポート番号が誤っている
Publicly Accessible 外部接続設定が無効
VPC設定 サブネットやルート設定の問題
DB停止 RDSインスタンスが停止状態

2. 調査手順

2-1. RDSステータスを確認する

AWSコンソールからRDSインスタンスが Available 状態であることを確認します。 


aws rds describe-db-instances

2-2. Security Groupを確認する

最も多い原因がSecurity Group設定ミスです。

  • 3306（MySQL）
  • 5432（PostgreSQL）
  • 1433（SQL Server）

接続元サーバーまたはIPアドレスが許可されているか確認してください。

2-3. 疎通確認を行う


telnet rds-endpoint.amazonaws.com 3306
        

nc -vz rds-endpoint.amazonaws.com 3306

接続できなければネットワーク設定に問題があります。

3. Spring Boot設定例

application.propertiesの設定例です。


spring.datasource.url=jdbc:mysql://sample-db.xxxxx.ap-northeast-1.rds.amazonaws.com:3306/sampledb

spring.datasource.username=admin

spring.datasource.password=password

spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver

エンドポイント、ポート番号、データベース名を確認してください。


@RestController
public class HealthController {

  @GetMapping("/health")
  public String health() {
    return "OK";
  }

}

4. よくあるエラー

Communications link failure


Communications link failure

RDSへネットワーク接続できていません。

  • Security Group確認
  • VPC確認
  • ポート確認

Connection refused


Connection refused

ホストまたはポート番号が誤っている可能性があります。

Access denied for user


Access denied for user

DBユーザーまたはパスワードが間違っています。

Unknown database


Unknown database

JDBC URL内のデータベース名を確認してください。

5. まとめ

  • まずRDSが起動中か確認する
  • Security Groupを確認する
  • JDBC URLを確認する
  • 認証情報を確認する
  • telnetやncで疎通確認する

実務ではRDS接続障害の約8割がSecurity Groupまたは接続文字列設定ミスです。最初にこの2点を確認すると効率よく原因を特定できます。
