1. Elastic Beanstalkデプロイ失敗の主な原因

Elastic Beanstalkのデプロイ失敗は、アプリケーションの不具合よりも設定ミスが原因であることが多くあります。

原因 発生頻度 影響
ポート設定ミス 高 起動失敗
jar作成失敗 高 デプロイ不可
環境変数未設定 高 DB接続失敗
ヘルスチェック失敗 高 環境異常判定
メモリ不足 中 起動途中で停止

2. デプロイ前の確認項目

2-1. jarファイル作成確認

Spring Bootアプリが正常にビルドできることを確認します。


mvn clean package

targetフォルダにjarファイルが生成されている必要があります。

2-2. ローカル起動確認


java -jar app.jar

ローカルで起動できないアプリはElastic Beanstalkでも起動できません。

2-3. DB接続情報確認

application.propertiesに固定値を書かず、環境変数から取得する構成が推奨です。 


spring.datasource.url=${DB_URL}
spring.datasource.username=${DB_USER}
spring.datasource.password=${DB_PASSWORD}

3. Spring Bootデプロイ手順

手順1：アプリケーション作成

Elastic Beanstalkコンソールから新規アプリケーションを作成します。

手順2：プラットフォーム選択

Javaプラットフォームを選択します。

手順3：jarアップロード

作成したjarファイルをアップロードします。

手順4：環境変数設定

Configuration → Softwareから設定します。

手順5：ログ確認

デプロイ失敗時はログを確認します。


eb logs

4. よくあるエラーと対処法

Application deployment failed


Application deployment failed

原因

  • jarファイル破損
  • ビルド失敗

対処法

  • 再ビルド実施
  • ローカル起動確認

Environment health has transitioned


Environment health has transitioned from Ok to Severe

原因

  • アプリ起動失敗
  • ヘルスチェック失敗

対処法

  • ログ確認
  • ポート設定確認

Port 8080 already in use


Port 8080 was already in use

原因

  • ポート競合

対処法


server.port=5000

Elastic Beanstalkの設定に合わせてポート番号を確認します。

Database connection failed

原因

  • RDS接続失敗
  • 環境変数未設定
  • セキュリティグループ設定ミス

対処法

  • RDS接続確認
  • 環境変数確認
  • ポート開放確認

5. まとめ

  • デプロイ失敗の多くは設定ミスが原因
  • まずローカルでjar起動確認を行う
  • 環境変数とDB接続設定を確認する
  • ヘルスチェックエラーはログから調査する
  • eb logsで詳細ログを確認する
