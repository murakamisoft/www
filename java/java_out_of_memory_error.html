OutOfMemoryErrorの原因と対処法

Javaアプリケーションで発生する「java.lang.OutOfMemoryError」。メモリ不足を示す深刻なエラーですが、原因はメモリリークから単なるヒープサイズ不足まで多岐にわたります。本記事では、このエラーの調査方法と根本的な改善策を解説します。

1. 概要

OutOfMemoryErrorは、JVMが新しいオブジェクトを割り当てるためのメモリ（ヒープ領域など）を確保できないときに発生します。放置するとアプリケーションのクラッシュに直結するため、迅速な調査が必要です。

項目 内容
環境 Java 21
主な調査ツール jmap, jstack, VisualVM, Eclipse MAT

2. 手順

2-1. ヒープダンプの取得

エラー発生時に自動的にダンプを取得する設定をJVMオプションに追加しておくと調査が捗ります。 


# JVM起動オプションに追加
-XX:+HeapDumpOnOutOfMemoryError -XX:HeapDumpPath=/path/to/dumps/

2-2. メモリリークの分析

取得したダンプファイルを「Eclipse Memory Analyzer (MAT)」などのツールで解析し、どのオブジェクトがメモリを占有しているか特定します。

3. サンプルコード（設定の指針）

物理メモリが許す範囲で、適切なヒープサイズを割り当てます。


# Xms（初期サイズ）とXmx（最大サイズ）を揃えるのが一般的
java -Xms2g -Xmx2g -jar my-application.jar

4. よくあるエラー

主な種類

  • Java heap space: オブジェクト作成数が多すぎるか、メモリリークの可能性。
  • Metaspace: クラス定義が多すぎる（ライブラリの動的ロードなど）。
  • GC overhead limit exceeded: GCが長時間かかっているのにメモリが解放されない状態。

5. まとめ

  • エラー発生時にヒープダンプを取得できるようにJVM設定を行う。
  • ツールを使用してメモリを圧迫しているオブジェクトを特定する。
  • プログラムの不要な参照（static変数への保持など）を見直す。
