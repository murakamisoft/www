UnsupportedOperationExceptionとは

Javaで要素を追加・削除しようとした瞬間にプログラムが停止するUnsupportedOperationException。これは主に「変更不可（イミュータブル）」なコレクションを操作しようとした際に発生します。なぜ発生するのか、どう修正すべきかを解説します。

1. 概要

UnsupportedOperationExceptionは、呼び出されたメソッドがサポートされていない（実装されていない）場合にスローされる非チェック例外です。Javaでは、固定されたコレクションに対する変更操作を行った際によく遭遇します。

項目 内容
主な要因 変更不可コレクションへのadd/remove操作
環境 Java 21 / Spring Boot 3.x

2. なぜ発生するのか

Java 9以降で導入されたList.of()やSet.of()、あるいはArrays.asList()（サイズ固定）など、特定のメソッドで作成されたコレクションは、要素の変更が許可されていません。これに対してadd()等を呼ぶと、例外が発生します。

3. サンプルコード

変更したい場合は、新しいミュータブル（変更可能）なインスタンスにコピーする必要があります。 


// NG: 変更不可のリスト
List<String> list = List.of("A", "B");
list.add("C"); // ここで UnsupportedOperationException

// OK: 変更可能なリストにコピーして使用する
List<String> mutableList = new ArrayList<>(List.of("A", "B"));
mutableList.add("C"); // 成功

4. よくあるエラー

スタックトレース例


java.lang.UnsupportedOperationException
    at java.base/java.util.ImmutableCollections.uoe(ImmutableCollections.java:...)
    at java.base/java.util.ImmutableCollections$AbstractImmutableCollection.add(...)

ImmutableCollectionsという文字が見えたら、それは「変更できないコレクションを操作しようとした」ことが原因です。

5. まとめ

  • 原因は「変更不可」なコレクションに対して変更操作を行っていること。
  • List.of() 等で作られたリストはイミュータブルであることを意識する。
  • 要素を変更する必要がある場合は、new ArrayList<>(...) で包んで操作する。
