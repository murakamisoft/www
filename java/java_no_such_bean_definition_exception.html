NoSuchBeanDefinitionExceptionの原因と解決方法

Spring Bootアプリケーション起動時に「No qualifying bean of type...」というエラーで停止していませんか？この記事では、DIコンテナがBeanを認識できない主な原因と、それを解消するための設定確認手順をステップバイステップで解説します。

1. 概要

NoSuchBeanDefinitionExceptionは、Springが依存性注入（DI）を行おうとした際に、指定された型や名前のBeanがコンテナ内に登録されていない場合に発生する例外です。

項目 内容
発生時期 アプリケーション起動時 / Bean取得時
主な要因 @Component等の付与漏れ、ComponentScanの範囲外
環境 Java 21 / Spring Boot 3.x

2. 主な原因と確認手順

2-1. アノテーションの確認

クラスに @Component, @Service, @Repository, @Controller などのアノテーションが付いているか確認してください。

2-2. パッケージ構成の確認

Spring Bootは、メインクラス（@SpringBootApplicationが付いたクラス）と同じか、そのサブパッケージ配下しかスキャンしません。スキャン対象外のパッケージにBeanを配置していないか確認してください。

3. コンポーネントスキャンの設定

どうしても別のパッケージにあるBeanを読み込みたい場合は、メインクラスでスキャン範囲を明示的に指定します。 


@SpringBootApplication
@ComponentScan(basePackages = {"com.example.app", "com.external.library"})
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}

4. よくあるエラー

エラーメッセージ例


org.springframework.beans.factory.NoSuchBeanDefinitionException: 
No qualifying bean of type 'com.example.MyService' available

対処法:
このメッセージが出たら、MyServiceクラスが本当に管理対象（Bean）になっているか、あるいはオートワイヤリング（@Autowired）する際に型の指定が間違っていないかを確認してください。

5. まとめ

  • Beanクラスにアノテーションが付いているか再確認する。
  • メインクラスの配下にクラスがあるか、パッケージ構成をチェックする。
  • スキャン対象外の場合は @ComponentScan で範囲を追加する。
