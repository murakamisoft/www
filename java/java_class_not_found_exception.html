ClassNotFoundExceptionの原因と解決方法

Java開発中に発生する「ClassNotFoundException」に頭を抱えていませんか？この記事では、このエラーが発生する主な原因から、クラスパスの確認、ライブラリの依存関係トラブルの解決手順までを丁寧に解説します。

1. 概要

ClassNotFoundExceptionは、Javaの実行時にJVM（Java仮想マシン）が指定された名前のクラスを見つけることができない場合に発生する例外です。主にアプリケーションの起動時や、特定のライブラリをロードしようとしたタイミングで発生します。

項目 内容
例外の種類 Checked Exception
主な要因 クラスパス未設定、jarファイルの不足、クラス名のタイプミス
推奨環境 Java 17以降推奨

2. 主な原因と確認手順

2-1. クラスパス（Classpath）の確認

最も多い原因は、実行時にクラスパスが正しく通っていないことです。コマンドラインで実行する場合、-cpまたは-classpathオプションを確認してください。 


# クラスパスを指定して実行する例
java -cp .:lib/my-library.jar com.example.Main

2-2. 依存関係（Dependency）の確認

MavenやGradleを使用している場合、ライブラリが正しくダウンロードされていないか、pom.xml / build.gradleでの指定漏れが考えられます。 


<dependency>
    <groupId>org.example</groupId>
    <artifactId>example-lib</artifactId>
    <version>1.0.0</version>
</dependency>

3. クラスパス確認のサンプルコード

現在実行中の環境で、どのクラスパスが読み込まれているかを確認するためのJavaコード例です。 


public class ClasspathChecker {
    public static void main(String[] args) {
        String classpath = System.getProperty("java.class.path");
        System.out.println("現在のクラスパス:");
        for (String path : classpath.split(System.getProperty("path.separator"))) {
            System.out.println(path);
        }
    }
}

4. よくあるエラーとその対処法

スタックトレースの例


java.lang.ClassNotFoundException: com.example.MyClass
    at java.base/jdk.internal.loader.BuiltinClassLoader.loadClass(...)
    ...

対処法:

  • クラス名の確認: パッケージ名を含めて正確に記述されているか確認してください。
  • Jarファイルの配置: 必要なJarファイルが正しいディレクトリに配置されているか再確認してください。
  • IDE設定: EclipseやIntelliJを使用している場合、Build Path設定からライブラリが正しく追加されているか確認してください。

5. まとめ

  • ClassNotFoundExceptionは、JVMがクラスを見つけられないときに発生する。
  • まずはクラスパスの設定が正しいかを最優先で確認する。
  • ビルドツール（Maven/Gradle）を使用している場合は、依存関係の宣言にミスがないかチェックする。
