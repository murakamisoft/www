IndexOutOfBoundsExceptionの原因と解決方法

Javaプログラミングで配列やListを扱う際によく遭遇する「IndexOutOfBoundsException」。存在しないインデックスにアクセスしようとした際に発生するこのエラーの原因と、安全なコードの書き方を詳しく解説します。

1. 概要

IndexOutOfBoundsExceptionは、配列やリストの有効範囲を超えたインデックス（負の数やサイズ以上の数）を指定して要素を取得・設定しようとした際に投げられる実行時例外です。

項目 内容
例外の種類 RuntimeException（非チェック例外）
主な発生場所 配列の添え字アクセス、List.get(index)
環境 Java 21 / Spring Boot 3.x

2. 原因の特定

2-1. ループ条件のミス

最も多いのは、ループの終了条件を i <= list.size() のように「イコール」を含めてしまうミスです。インデックスは 0 から size - 1 までであることに注意してください。

2-2. 空のリストへのアクセス

要素が一つも入っていないリストに対して list.get(0) を実行すると、当然ながらこの例外が発生します。

3. サンプルコード

安全にリストの全要素を走査する方法です。拡張for文を使用すれば、インデックスを意識する必要がないため、このエラーを確実に防げます。 


import java.util.List;

public class Sample {
    public void printList(List<String> items) {
        // 安全な拡張for文
        for (String item : items) {
            System.out.println(item);
        }
    }
}

4. よくあるエラー

例外の出力例


Exception in thread "main" java.lang.IndexOutOfBoundsException: Index 3 out of bounds for length 3

「Index 3」へアクセスしようとしたが、実際の長さは「3」しかない（0, 1, 2までしか使えない）というメッセージです。配列やListの長さを確認し、添え字が範囲内に収まっているかデバッグを行ってください。

5. まとめ

  • インデックスは「0」から始まることを再確認する。
  • ループ条件には「<（未満）」を使用し、「<=（以下）」は避ける。
  • 可能な限り拡張for文やStream APIを利用してインデックス管理を避ける。
