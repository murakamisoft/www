NumberFormatExceptionの原因と対処法

Javaで文字列を数値（IntegerやLongなど）に変換しようとした際、形式が正しくないと発生するNumberFormatException。この例外が発生する仕組みと、ユーザー入力値などを安全に扱うためのベストプラクティスを解説します。

1. 概要

NumberFormatExceptionは、文字列を数値に変換するメソッド（Integer.parseInt()など）に対して、数値として解釈できない文字列が渡されたときに発生する非チェック例外です。

項目 内容
主な発生場所 Integer.parseInt, Long.parseLong, Double.parseDouble
発生原因 空文字、空白文字、数値以外の文字（アルファベットなど）の混入
環境 Java 21 / Spring Boot 3.x

2. 原因と発生パターン

2-1. 不正な文字列の変換

数値型ではない「abc」や「12.34（整数型への変換時）」を変換しようとすると例外になります。

2-2. nullまたは空文字

nullを渡すとNullPointerExceptionですが、空文字「""」を変換しようとした場合も同様にNumberFormatExceptionが発生します。

3. 安全な数値変換の方法

try-catchブロックで囲むか、あらかじめ正規表現でチェックを行うのが定石です。 


public Integer convertToInteger(String input) {
    try {
        return Integer.parseInt(input);
    } catch (NumberFormatException e) {
        // ログ出力やデフォルト値を返すなどのハンドリング
        System.err.println("不正な値です: " + input);
        return 0;
    }
}

4. よくあるエラー

例外のスタックトレース例


java.lang.NumberFormatException: For input string: "123a"
    at java.base/java.lang.NumberFormatException.forInputString(...)

「For input string: "123a"」と出ている通り、この文字列が変換対象として不適切であることを示しています。フォームの入力値など、外部から渡ってくるデータには必ずバリデーションをかけましょう。

5. まとめ

  • 文字列を数値変換する際は、常に変換失敗の可能性を考慮する。
  • try-catchで例外をキャッチし、適切にハンドリングする。
  • 入力フォームなどで値を受け取る場合は、変換前に正規表現等で数値であることを確認する。
