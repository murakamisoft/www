StackOverflowErrorの原因と解決方法

プログラムの実行中に突然停止し、「java.lang.StackOverflowError」が出力されたことはありませんか？これはスレッドのスタック領域が枯渇したことを示すエラーです。なぜ発生するのか、そのメカニズムと解決のヒントを解説します。

1. 概要

StackOverflowErrorは、JVMのスタック領域（メソッドの呼び出し履歴やローカル変数が格納される領域）が不足した際に発生するErrorです。多くの場合、コードのロジックに問題があります。

項目 内容
例外の種類 Error（例外ではなく回復不能なエラー）
主な要因 無限再帰、深い階層のメソッド呼び出し
環境 Java 21 / Spring Boot 3.x

2. 原因の特定方法

2-1. スタックトレースを確認

エラーログを確認すると、同じメソッドが数百～数千回も連続して呼び出されている箇所が見つかるはずです。そこが「無限再帰」の起点です。

2-2. 呼び出し階層の検討

再帰構造が論理的に正しいか、終了条件（ベースケース）が正しく設定されているかを確認します。

3. サンプルコード

問題のある再帰と、それを解決するループ処理の比較です。


// 問題があるコード（無限再帰）
public void recursiveMethod() {
    recursiveMethod(); // 終了条件がないため無限ループへ
}

// 解決後のコード（ループ処理）
public void loopMethod() {
    for (int i = 0; i < 100; i++) {
        // 安全な反復処理
    }
}

4. よくあるエラー

エラーの出力例


Exception in thread "main" java.lang.StackOverflowError
    at com.example.MyClass.recursiveMethod(MyClass.java:10)
    at com.example.MyClass.recursiveMethod(MyClass.java:10)
    ...

上記のように同じファイル・同じ行番号が延々と続く場合は、再帰呼び出しがループしています。終了条件を必ず設定してください。

5. まとめ

  • 再帰を使用する場合は、必ず「終了条件」を設ける。
  • 深い再帰が必要な場合は、ループ（for/while）への書き換えを検討する。
  • どうしても深い再帰が必要な場合は、JVMオプション（-Xss）でスタックサイズを増やすが、根本解決を優先する。
