ConcurrentModificationExceptionとは

Javaでコレクションをループ（拡張for文など）で回している最中に、そのコレクションに対して要素の追加や削除を行うと、ConcurrentModificationExceptionが発生します。なぜこの例外が起きるのか、どうすれば安全に処理できるのかを解説します。

1. 概要

ConcurrentModificationExceptionは、コレクションが「フェイルファスト（fail-fast）」動作を求められた際に発生する例外です。ループ中のデータ構造の整合性が崩れることを防ぐために投げられます。

項目 内容
発生タイミング イテレータを使用中にコレクションが変更された時
主な対象 ArrayList, HashMap, HashSetなど
環境 Java 21 / Spring Boot 3.x

2. 原因と発生パターン

2-1. 拡張for文での削除

最も一般的なミスは、拡張for文の中でlist.remove()を呼び出すことです。 


List<String> list = new ArrayList<>(List.of("A", "B", "C"));
for (String s : list) {
    if (s.equals("B")) {
        list.remove(s); // ここで例外が発生
    }
}

2-2. なぜエラーになるのか

Javaの内部イテレータは、コレクションがループ中に変更されたことを検知すると、矛盾を防ぐために即座に例外を投げます。

3. 安全な回避方法

安全に要素を削除するには、Iteratorを使用するか、Java 8以降のremoveIfメソッドを使用するのが最適です。 


// おすすめ：removeIfメソッド（Java 8以降）
List<String> list = new ArrayList<>(List.of("A", "B", "C"));
list.removeIf(s -> s.equals("B"));

4. よくあるエラー

エラーメッセージ例


Exception in thread "main" java.util.ConcurrentModificationException
    at java.base/java.util.ArrayList$Itr.checkForComodification(ArrayList.java:...)

このスタックトレースが出た場合は、ループ内でのリスト操作（add, remove）を疑ってください。

5. まとめ

  • ループ中にコレクションを直接変更してはいけない。
  • Java 8以降なら removeIf を使うのが最も簡単。
  • 複雑な操作が必要な場合は Iterator.remove() を明示的に呼び出す。
